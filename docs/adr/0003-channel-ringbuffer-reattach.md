# Channel + 环形缓冲 PTY 重连

Status: Accepted

PTY 输出的 IPC 通道使用可替换的 Tauri `Channel` + 256KB 环形缓冲。前端页面切换时断开当前 Channel，后台继续写环形缓冲；目标页面挂载时替换 Channel 并回放缓冲内容。

**为什么不是全量缓冲？** 内存无界增长。终端可能运行数小时产生 GB 级输出。

**为什么不是 VT 缓冲回放（xterm.js serialize）？** xterm.js 的 `serialize()` 返回已解析的终端画面格子，但控制字符（颜色、光标移动）已被终端状态机消费，无法还原。环形缓冲保留原始字节流，重放时 xterm.js 完整解析。

**备选方案**：页面切换时杀进程重建（用户体验差，丢失运行状态）；单页面架构（限制死，不支持多项目工作流）。

**后果**：最大可恢复滚动历史为 256KB。环形缓冲淘汰按行边界 1KB 粒度——不会截断半个转义序列导致终端状态损坏。
