# 侧栏改造需求：活动栏 + 可关闭侧栏视图

> 版本：v1.0（2026-07-18） · 状态：需求已定稿，待制定开发计划
> 相关文档：术语定义见根目录 `CONTEXT.md`「侧栏」一节；核心设计权衡见 `.claude/adr.md` ADR-0001

## 1. 背景与目标

当前 slTerminal 主界面为 Allotment 常驻三栏：

```
┌────────────┬────────────┬──────────────────────────┐
│ 项目列表    │ 文件浏览器  │ 主区（Dockview 操作页面） │
│ 250px 常驻  │ 250px 常驻  │                          │
└────────────┴────────────┴──────────────────────────┘
```

项目列表（`SidebarTree`）与文件浏览器（`ExplorerPanel`）**常驻且不可关闭**，合计约占 500px 宽度，挤压主区。

本次改造目标：

1. 两个栏目变为**可关闭**的视图，用户需要时才打开；
2. 最左侧新增**活动栏**作为开关入口（VS Code 风格）；
3. 两视图**共享**同一块展示区域，支持上下分区同时展示；
4. 架构上支持未来低成本新增更多同类视图（高内聚、低耦合、易扩展）。

## 2. 术语

以下术语已与项目 `CONTEXT.md` 对齐，本文档及后续代码、文档统一使用：

| 术语 | 英文 | 定义 |
|------|------|------|
| **活动栏** | Activity Bar | 应用最左侧的窄条（约 40px 宽），常驻不可关闭，容纳侧栏视图按钮。按钮可通过鼠标左键拖拽在上区/下区之间移动 |
| **侧栏视图** | Side View | 活动栏按钮对应的可开关内容视图。当前有两个：项目列表视图、文件浏览器视图。**禁止**称为"页面"（与操作页面冲突）或"面板"（与 Dockview 面板冲突） |
| **项目列表** | — | 项目/操作页面二级树（现 `SidebarTree` 组件），管理项目 CRUD、操作页面 CRUD、页面切换导航 |
| **文件浏览器** | — | 活跃项目的文件树（现 `ExplorerPanel` 组件），支持文件 CRUD、git 状态着色、文件变更增量刷新 |
| **侧栏区** | Side Bar | 侧栏视图共享的展示区域，位于活动栏与主区之间，垂直划分为上区和下区两个半区 |
| **上区 / 下区** | Top Zone / Bottom Zone | 侧栏区的两个半区。每个活动栏按钮经拖拽归属于其一，决定对应视图的展示位置 |

## 3. 改造后的布局结构

```
┌─────┬───────────────┬──────────────────────────┐
│     │   侧栏区       │                          │
│ 活  │ ┌───────────┐ │                          │
│ 动  │ │   上区     │ │  主区（Dockview          │
│ 栏  │ │ (视图槽位) │ │   操作页面，逻辑不变）    │
│     │ ├───────────┤ │                          │
│ 📋  │ │   下区     │ │                          │
│ 📁  │ │ (视图槽位) │ │                          │
│     │ └───────────┘ │                          │
│~40px│  160–500px    │                          │
└─────┴───────────────┴──────────────────────────┘
```

- **活动栏**：固定窄条（约 40px），始终可见，不可关闭。内含视图按钮，分上、下两个按钮组（对应上区/下区归属）。
- **侧栏区**：原两栏合并为一栏，保留与主区之间的 Allotment 水平分隔条，可拖拽调宽，范围 160–500px。内部垂直嵌套 Allotment，上区/下区之间有可拖拽分隔条调节高度比例。**当没有任何打开的视图时，侧栏区整体隐藏**（见 §5）。
- **主区**：现有 Dockview 多实例架构完全不变。

## 4. 核心状态机（最重要的规则）

### 4.1 状态模型

系统只需维护两类事实，全部展示由它们**纯推导**，无任何额外 UI 临时态：

1. **按钮归属**：每个视图按钮归属于上区或下区（含半区内排列顺序）；
2. **各半区打开的视图**：每半区一个槽位，要么为空、要么为某个视图 id。

**关键不变量：每个半区最多只有一个"打开"的视图。不存在"打开但未展示"的中间状态，不需要也不允许任何历史记忆。**

### 4.2 规则表

| # | 操作 | 结果 |
|---|------|------|
| R1 | 点击**未打开**的按钮 | 该按钮所在半区打开对应视图；若该半区已有打开的视图，**隐式关闭它**（新视图取而代之） |
| R2 | 点击**已打开**的按钮 | 关闭该视图（槽位置空） |
| R3 | 两个半区均为空 | **侧栏区整体隐藏**，活动栏直接贴主区，主区占满剩余宽度 |
| R4 | 仅一个视图打开 | 该视图**全高展示**（占满整个侧栏区，不显示上下分割） |
| R5 | 两个视图分别在两半区打开 | 上下分开展示：上半区展示上区视图，下半区展示下区视图，中间为可调分隔条 |
| R6 | 拖拽按钮**跨半区**（其视图打开中） | 视图**立即跟随**移动到新半区展示；若目标半区已有打开的视图，**关闭那个视图**（同 R1 的隐式关闭） |
| R7 | 拖拽按钮**跨半区**（其视图未打开） | 仅改变归属，展示不发生任何变化 |
| R8 | 拖拽按钮**半区内**移动 | 改变按钮在该半区的排列顺序（拖拽落点即插入位置）；跨区拖拽同样按落点插入目标半区的对应位置 |
| R9 | 注册新视图（未来扩展） | 按钮默认追加到**上区末尾**，初始为关闭状态 |

### 4.3 展示推导示例

设"打开集合"为当前各半区槽位的内容：

| 上区槽位 | 下区槽位 | 侧栏区展示 |
|---------|---------|-----------|
| 空 | 空 | 整体隐藏 |
| 项目列表 | 空 | 全高展示项目列表 |
| 空 | 文件浏览器 | 全高展示文件浏览器 |
| 项目列表 | 文件浏览器 | 上半：项目列表；下半：文件浏览器（比例可调） |

## 5. 场景走查（验收用例）

以下场景全部可从 §4 规则表推导，作为实现后的行为验收标准。记按钮：P=项目列表（📋）、E=文件浏览器（📁）。

**S1（基础开关，对应原始需求规则 3）**——初始 P、E 均在上区，均关闭：
1. 点击 P → 全高展示项目列表；
2. 点击 E → E 在上区打开，P 被**隐式关闭**，全高展示文件浏览器（R1）；
3. 再次点击 E → E 关闭，两半区均空，**侧栏区整体隐藏**（R2+R3）。**注意：不是重新展示项目列表**——无历史记忆。

**S2（双视图分区，对应原始需求规则 4）**——P 在上区、E 在下区：
1. 点击 P → 全高展示项目列表（R4，注意不是只在上半展示）；
2. 点击 E → 上半展示项目列表、下半展示文件浏览器（R5）；
3. 再次点击 P → P 关闭，**全高展示文件浏览器**（R4，注意不是只在下半展示）。

**S3（同区替换）**——P、E 均在上区，P 打开中：
1. 点击 E → E 打开、P 关闭，全高展示文件浏览器（R1）；
2. 再点击 E → 侧栏区整体隐藏。

**S4（拖拽换区 + 视图跟随）**——P 在上区且打开中，E 在下区且打开中（上下分区展示）：
1. 将 P 按钮从上区拖到下区 → 项目列表立即移到下区展示；下区原有的文件浏览器被**关闭**（R6）；此时仅项目列表一个视图打开 → 全高展示（R4）。

**S5（拖拽未打开按钮）**——P 在上区打开中，E 在下区关闭中：
1. 将 E 按钮拖至上区 → 仅归属变化，展示不变，仍全高展示项目列表（R7）；
2. 点击 E → E 在上区打开，P 被关闭（R1）。

**S6（半区内排序）**——未来上区有按钮 A、B、C：
1. 拖拽 C 到 A 前方 → 顺序变为 C、A、B，各视图开关状态不受影响（R8）。

## 6. 外观规范

| 项 | 规范 |
|----|------|
| 活动栏宽度 | 约 40px，固定，常驻不可关闭 |
| 按钮图标 | Emoji：**项目列表 📋**、**文件浏览器 📁**（两者辨识度需高，不得使用 🗂️ 这类与 📁 形近的图标）；未来新视图在注册时配置 |
| 按钮状态指示 | VS Code 风格：打开中的按钮高亮 + 左侧 2px 指示条；鼠标悬停有 hover 反馈；tooltip 显示视图名称（如"项目列表"） |
| 按钮组分隔 | 上区按钮组与下区按钮组之间视觉上分隔（下区组固定在活动栏底部） |
| 配色 | 所有颜色引用 `theme/colors.ts` token（硬约束 #6），禁止硬编码色值；仅暗色模式 |
| 拖拽反馈 | 拖拽过程中显示插入位置指示（落点线）；拖拽仅限活动栏内（上区 ↔ 下区 + 半区内排序），拖出活动栏无效 |

## 7. 默认状态（首次启动 / 无持久化数据）

- 项目列表 📋、文件浏览器 📁 两个按钮均在**上区**，项目列表在前；
- **项目列表默认打开**，文件浏览器默认关闭；
- 侧栏区默认展示项目列表（理由：项目列表是项目/页面导航的唯一入口；没有活跃项目时文件浏览器为空树，无引导价值）。

## 8. 持久化

以下四项状态写入 `~/.slterminal/settings.json` 新增的 **`sideBar`** 段，**重启后恢复现场**：

| 持久化项 | 说明 |
|---------|------|
| 按钮归属 + 半区内顺序 | 如 `{ top: ["projects", "explorer"], bottom: [] }` |
| 侧栏区宽度 | px，关闭再打开、重启后恢复 |
| 上区/下区分割比例 | 双视图分区展示时生效 |
| 各半区当前打开的视图 | 如 `{ top: "projects", bottom: null }`；重启后恢复打开状态 |

**实现约束**：复用现有 `settings.rs` 后端浅合并机制（`save_settings` 只写 `sideBar` slice，不擦除 `fontSize`、`keybindings` 等其他段），零后端改动；前端新增 `stores/sideBar.ts`，与 `fontSize.ts` / `keybindings.ts` 同模式（`loaded` 守卫 + 2s debounce 保存 + loadFromDisk sanitize）。

## 9. 关闭语义：隐藏不卸载

- 关闭视图 = 组件 `display:none` **隐藏，不卸载**。视图内部状态（如文件浏览器已展开的目录、滚动位置）完整保留，再次打开时原样呈现。
- 与操作页面多实例显隐（H6 方案）同一模式，架构一致。
- 隐藏期间文件浏览器的 fs-event 订阅继续生效，重新显示时数据为最新（优点而非开销）。

## 10. 明确不做（非目标）

| 项 | 说明 |
|----|------|
| ❌ 快捷键 | 不为活动栏/侧栏视图添加任何快捷键（包括 Ctrl+B），用户明确否决 |
| ❌ 历史记忆 | 关闭视图后不回到"前一个打开的视图"，无打开历史栈 |
| ❌ "打开未展示"态 | 每半区严格单槽位，被替换的视图即关闭，不存在隐藏保留 |
| ❌ 按钮隐藏/右键菜单 | 活动栏按钮注册即显示，不支持用户隐藏按钮，无右键菜单 |
| ❌ 拖出活动栏 | 拖拽仅用于上区 ↔ 下区换区与半区内排序，不支持把按钮拖到主区等其他区域 |
| ❌ 主区改动 | Dockview 操作页面、页签、面板系统完全不动 |

## 11. 扩展性设计（SideViewRegistry）

新增侧栏视图的成本必须低至"实现组件 + 一行注册"。采用**模块级注册表单例**模式——与项目现有四个注册表（`panelRegistry`、`TabTitleRegistry`、`FileViewerRegistry`、`ShortcutRegistry`）同一模式：

```typescript
// 概念示意（接口以开发计划为准）
sideViewRegistry.register({
  id: "explorer",            // 唯一 id（持久化引用）
  title: "文件浏览器",        // tooltip / 无障碍名称
  icon: "📁",                // 活动栏按钮图标
  component: ExplorerPanel,  // 视图组件
});
```

注册后以下能力**自动生效**，无需改动框架代码：活动栏按钮渲染与开关、上区/下区拖拽归属与排序、侧栏区槽位展示、全部持久化项。

**扩展步骤**（未来新增视图，如"搜索视图"）：

1. 在 `src/features/<name>/` 实现视图组件；
2. 在注册文件中追加一行 `sideViewRegistry.register({...})`；
3. 按钮自动出现在上区末尾（R9），默认关闭。

## 12. 对现有组件的影响

| 现有件 | 变化 |
|--------|------|
| `SidebarTree`（项目列表） | 组件本体**不变**，仅宿主从 Allotment 常驻栏变为侧栏区视图槽 |
| `ExplorerPanel`（文件浏览器） | 同上，组件本体**不变** |
| `Workspace.tsx` | Allotment 三栏改为：活动栏（固定）+ 侧栏区（可显隐、内含垂直分区）+ 主区；视图显隐由 sideBar store 驱动 |
| 主区 / Dockview / 页签标题 | **完全不变** |
| `settings.rs`（后端） | **不变**（复用浅合并） |
| E2E helpers | `__slterm_e2e_createProject` 等依赖 SidebarTree 渲染的路径需保证项目列表视图默认打开或测试中显式打开（细节在开发计划中确定） |
