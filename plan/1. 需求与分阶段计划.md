# Claude Code 终端模拟器 — 需求规格与分阶段开发计划

> 一个面向 Windows 10/11、专门为 Claude Code CLI 调优的通用终端模拟器。本文档锁定需求,并给出”每步独立可运行、严格按序”的开发计划。

-----

## 一、定位与全局约束

- 通用终端模拟器,但围绕”同时跑多个不同项目的 Claude Code 会话”专门调优。
- 在 **Windows 原生**跑 `claude`(不走 WSL)。
- **单窗口**:一台 PC 只运行一个客户端实例,不做多窗口。
- **仅暗色模式**,不考虑浅色。
- 渲染层 **GPU 加速**(作为终端/编辑器渲染的基线属性)。
- 默认 shell:**PowerShell 7**。窗格起来后由用户自己敲 `claude` 或其它命令。
- 复制 = **Ctrl+Shift+C**(Ctrl+C 保留为中断,供 claude 取消用)。

-----

## 二、整体布局(从左到右 + 底部)

1. **主侧边栏**(三级树)
- L1 **项目**:用户在 File Explorer 手动选择的文件夹路径。
- L2 **主工作区** + **worktree**(若有,worktree 与主工作区并列)。
- L3 **操作页面**。
1. **次级侧边栏**:文件浏览器,展示当前选中工作区的文件,文件名按 git 状态着色。
1. **主交互区**:每个操作页面 = 一套独立的”统一分屏页签”工作区(终端与文件编辑器都是页签)。
1. **底部状态栏**:展示 git 分支 / diff、Python 虚拟环境。

-----

## 三、功能需求(按模块)

### 1. 页签 / 分屏系统(核心)

- 新建操作页面 → 展示空白页,页上有一个”打开终端”按钮。
- 点”打开终端” → 开一个 PowerShell 7 终端页签;双击文件 → 开一个文件编辑器页签。
- “全屏” = 占满该操作页面的内容区(单页签最大化),非操作系统级全屏。
- 终端与文件编辑器统一为页签:已有页签时,新建终端 / 打开文件都作为新页签,加到**焦点页签所在页签栏**之后并展示。
- **拖拽分屏**:把页签拖到光标当前**悬停的分组**上,按光标靠近的边(上/下/左/右)决定分屏方向 —— 业界标准落点规则(参照 IntelliJ / VS Code,VS Code 本身就支持终端作为编辑区页签)。
- 支持**任意嵌套分屏**;某分组被拖空后自动合并回去;拖动分隔条调整各分屏比例。
- 每个操作页面拥有**独立**的分屏页签布局;切换操作页面整套切换;关掉最后一个页签回到”打开终端”空白页。
- 新建的 PowerShell 终端 cwd 到当前工作区路径。

### 2. 终端

- 后端 PowerShell 7;基础 PTY:输入/输出、resize、颜色、光标、重绘。
- 需能正确渲染 claude 的交互式 TUI(Ctrl+C 取消、Shift+Tab 切 plan mode、多行输入等)。
- 每个终端页签带一个**页签级活动状态点**(详见”Claude Code 集成”)。
- 滚动回看 + 查找。

### 3. 侧边栏与项目结构

- L1 项目:手动通过 File Explorer 添加文件夹路径。
- L2:项目根即主工作区;根目录有 git 则自动识别 git 管理。
- L3 操作页面:在项目下新建/管理;每个操作页面一套独立分屏布局。
- “项目快速切换” = 在侧边栏跳到某个已打开的项目/操作页面,或在指定路径下新开。

### 4. worktree

- Claude Code 自带 `--worktree`,但模拟器也提供一个 **worktree 管理区**(主侧边栏内,按项目)。
- L2 中 worktree 与主工作区并列展示(来源 `git worktree list`)。
- **新建**:模拟器执行 `git worktree add`(可命名)→ 自动开一个**新操作页面**,其终端是 cwd 到该 worktree 目录的 PowerShell;用户自己敲 claude。
- **绑定规则**:页签按所在目录归属 worktree;一个页签只对应一个 worktree,一个 worktree 可对应多个页签/操作页面;可在已有 worktree 上新开页签。
- **删除**:从管理区删;若该 worktree 还有打开的页签,先警告 + 二次确认,确认后关掉这些页签。
- 文件浏览器中隐藏 `.claude/worktrees/`,避免 worktree 在 L2 与文件夹里重复出现。

### 5. 文件浏览器(次级侧边栏)

- 根目录跟随当前选中的工作区(主工作区或某个 worktree)。
- 文件名按 git 状态着色(见”配色参考”)。
- 初期仅”查看 + 打开”,不做新建/重命名/删除。
- 双击文件 → 在**焦点操作页面**里作为页签打开。

### 6. 编辑器(文件作为统一页签)

- 可直接编辑 + 语法高亮 + 行内 git diff 边栏 + 保存(Ctrl+S)+ 基本查找(Ctrl+F)。
- **不在早期范围**:LSP、自动补全、跳转定义、重构。
- diff 边栏在编辑区每行最左侧用颜色标 git diff 状态,基线 = HEAD,配色见下。
- **与 claude 抢文件的处理**:文件被 claude 在外部改动时 —— 无未保存修改则自动重载;有未保存修改则弹窗(保留我的 / 重载 / 看差异)。注意:编辑器里未保存的内容 claude 看不到(它读磁盘上已保存版本)。其中"看差异"是"我的未保存内容 vs 磁盘新版本",与行内 diff 边栏(工作区 vs HEAD)不是同一视图,需单独的简易差异弹窗;若该弹窗暂未就绪,M3 先只给"保留我的 / 重载"两项。

### 7. git 集成

- 工作区根有 git 则自动识别。
- 底部状态栏:分支 + diff/dirty 概要。
- 文件浏览器:文件名状态着色。
- 编辑器:行内 diff 边栏。

### 8. Claude Code 集成(不把 claude 当黑盒,读其 hooks / 配置 / statusline)

- **活动状态**(经 hooks 锁存状态机驱动;事件名 / `notification_type` 经官方 hooks 文档核实):
  - 运行中:`UserPromptSubmit`(提交即触发,覆盖”思考中”窗口)→ `PreToolUse` / `PostToolUse`,**锁存**至 `Stop`,期间不按工具逐次翻转。
  - 需人工介入:`PermissionRequest` 或 `Elicitation`(MCP 取输入);兜底 `Notification`(`notification_type` = `permission_prompt` | `elicitation_dialog`)。
  - 等输入:`Notification`(`notification_type` = `idle_prompt`)。
  - 完成:`Stop`(主代理结束)。**`SubagentStop` 不算完成**——子代理结束时主代理仍在运行,维持”运行中”。
  - 其它生命周期:`SessionStart`(`source` = startup / resume / clear / compact)/ `PreCompact` / `SessionEnd`。
- **状态展示位置**:
  - 每个终端页签:页签级状态点。
  - 操作页面节点(L3):显示其中终端的聚合状态,优先级 **需介入 > 运行中 > 空闲**。
  - L1 / L2 不展示状态。
- **桌面通知**:仅 *需人工介入* 与 *执行完成* 两类;仅当对应终端页签未聚焦/不可见时弹;点击通知跳转到该页签。
- **token / 上下文% / 成本**:由模拟器**提供**一个 statusLine 脚本,claude 把会话 JSON 喂给它、它打印到 claude TUI 底部(字段经核实存在:`model`、`context_window.used_percentage`、`cost.total_cost_usd`、`rate_limits`)。模拟器**不另造组件、也不自行解析这些数字**(M5 仅显示在 claude TUI 内)。脚本须兜底:首个 API 调用前与 `/compact` 后 `current_usage` 为 null,且 `used_percentage` 仅计输入 token。

### 9. 底部状态栏

- git 分支、git diff/dirty 概要、Python 虚拟环境名。
- **venv 检测口径**:要显示"当前**已激活**的 venv/conda"(而非仅"目录是否有 `.venv`"),宿主进程读不到子 shell 的活动环境变量,须由注入的 PowerShell profile 随 prompt 发自定义 OSC 序列回传 `$env:VIRTUAL_ENV` / `$env:CONDA_DEFAULT_ENV`(见技术架构与 Phase 1 profile 注入)。

### 10. 键位 / 图片

- 自定义键位:**仅管终端/UI 自身动作**(新建/分屏/切换/复制等),其它按键透传给 claude。
- **图片粘贴**:剪贴板有图时,存临时文件 + 把文件路径**写入终端(`pty_write` 路径文本)**送给 claude(Windows 上最可靠);需验证 claude 在 Windows 对"纯路径文本"识别为图片(Phase 6 人工验收项)。

### 11. 会话与持久化

- 历史会话:每个项目列出 标题 / 时间,点击 `--resume`(在新操作页面/页签里续上)。
- **持久化范围**:跨重启持久化 **项目列表 / 各项目的操作页面结构(L3 名称)/ 设置 / 键位**;**不**持久化页签内容与 PTY 会话。
- 重启模拟器:**不**自动恢复任何页签,操作页面回到空白"打开终端"页(项目与操作页面结构仍在)。

### 12. 外观与渲染

- 仅暗色;主题/字体先用合理默认,后续可配置。
- 渲染 GPU 加速。

## 五、分阶段开发计划

> 原则:每一步都是一个**独立可运行/可演示**的小增量;**严格按序**;后面的步骤不阻塞前面的演示。

### M1 — 核心外壳(终端 + 基础编辑器 + 分屏)

1. 一个窗口 + 空的统一分屏页签区(暗色)。
1. 能新建一个 PowerShell 7 终端页签并交互(基础 PTY:输入/输出/resize)。
1. 能打开一个文件作为页签,直接编辑、保存(纯文本,暂无高亮/无 diff)。
1. 多页签:新建终端/打开文件作为新页签加到焦点页签后;页签切换、关闭;关掉最后一个回到”打开终端”空白页。
1. 拖拽分屏:页签拖到悬停分组的某条边,按业界落点规则分屏。
1. 任意嵌套分屏 + 拖空自动合并。
1. 拖动分隔条调整各分屏比例。
1. 能在终端里跑 claude 并正确渲染其 TUI(颜色/光标/重绘/Ctrl+C/Shift+Tab)。

### M2 — 侧边栏与项目结构

1. 主侧边栏 L1:手动经 File Explorer 添加项目并列出。
1. L3 操作页面:在项目下新建/管理;每个操作页面 = 一套独立 M1 布局,切换整套切换;新建时打开 PowerShell cwd 到项目路径。
1. L2 主工作区节点 + git 检测(根目录有 git 则识别)。
1. worktree 发现与展示(`git worktree list`,与主工作区并列)。
1. worktree 管理区 — 新建:`git worktree add` + 命名 → 新操作页面,终端 cwd 到该 worktree。
1. worktree 删除(有页签则警告 + 二次确认 + 关页签);落实页签↔worktree 目录绑定规则。

### M3 — 文件浏览器与编辑器增强

1. 次级侧边栏文件浏览器:展示当前工作区文件树;双击在焦点操作页面作为页签打开;隐藏 `.claude/worktrees/`。
1. 编辑器语法高亮。
1. 编辑器行内 git diff 边栏(HEAD 基线,暗色配色)。
1. 文件浏览器文件名 git 状态着色(暗色配色)。
1. 编辑器查找(Ctrl+F)。
1. 外部改动处理:干净则自动重载,脏则弹窗选择。

### M4 — git / Python 状态栏

1. 底部状态栏:git 分支。
1. 底部状态栏:git diff/dirty 概要。
1. 底部状态栏:Python 虚拟环境感知与展示。

### M5 — Claude Code 集成

1. 安装/读取 claude hooks,建立”会话 ↔ 终端页签”映射(经注入 PTY 的 `SLTERM_PANE_ID` 由 hook 回传精确归属)。
1. 终端页签级活动状态点(运行中/需介入/空闲…),由 hooks 驱动;思考窗口用 `UserPromptSubmit` 覆盖(不再近似为空闲)。
1. 操作页面节点聚合状态(需介入 > 运行中 > 空闲)。
1. 桌面通知:需人工介入 + 执行完成;仅对应页签未聚焦时弹;点击跳转。
1. statusline:提供/配置 statusLine 脚本,在 claude TUI 底部显示 上下文%/成本/额度。
1. 历史会话:列出每个项目 标题/时间,点击 `--resume`(新操作页面/页签)。

### M6 — 体验完善

1. 图片粘贴:剪贴板有图 → 存临时文件 + 路径经 `pty_write` 写入终端。
1. 自定义键位(仅终端/UI 动作,其它透传 claude)。
1. 暗色主题/字体可配置项。
1. 渲染/性能打磨(若 GPU 加速等未在 M1 一并解决)。

## 附:几个待你最终确认/可调整的点

- M2–M6 之间、以及每个 M 内部的步骤顺序可按你的偏好重排。
- 若希望某个能力(如图片粘贴、文件浏览器、statusline)更早出现,挪到靠前的 M 即可。
- 后续如需把编辑器升级为带 LSP 的完整 IDE 编辑,可作为 M3 之后的独立大阶段单列。
