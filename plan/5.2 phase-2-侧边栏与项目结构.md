# Phase 2 — 侧边栏与项目结构(开发文档)

> 目标:主侧栏三级(项目 → 主工作区+worktree → 操作页面),手动添加项目,git 检测,worktree 展示/新建/删除,操作页面 = 独立布局。

## 1. 目标与范围

- 交付:L1 项目分组 + L2 工作区/worktree + L3 操作页面;git 检测;worktree CRUD(建/列/删);页签↔worktree 目录绑定。
- 不在范围:文件浏览器(次级侧栏)、编辑器增强、状态栏、claude。

## 2. 前置依赖

- Phase 1(终端面板、Dockview 布局、layoutSerde、sessions store)。

## 3. 涉及的模块与文件

- 后端:`git/{mod.rs, worktree.rs}`、`fs/mod.rs`(选目录后读基本信息)。
- 前端:`features/sidebar/`、`features/worktree/`、`stores/{projects.ts, layout.ts}`、`workspace/layoutSerde.ts`、`ipc/git.ts`。

## 4. 开发内容(分任务)

1. L1 项目:系统文件夹选择器(Tauri dialog)添加路径 → `projects` store;侧栏按项目分组渲染。
1. git 检测:`git_is_repo(path) -> bool` / `git_root(path)`;是仓库则标记。
1. L2 工作区/worktree:`git_worktree_list(repo) -> [{name, path, branch}]`;主工作区 + worktree 并列展示。
1. L3 操作页面:工作区下”新建操作页面” → 新建一份空 Dockview 布局并打开 pwsh(cwd=工作区路径);切换操作页面 = `layoutSerde.fromJSON` 换布局;`stores/layout` 存每个操作页面的布局 JSON。
1. worktree 新建:`git_worktree_add(repo, name) -> {path}`(shell 出 `git worktree add`,落 `.claude/worktrees/<name>`、分支 `worktree-<name>`)→ 自动新建操作页面、终端 cwd 到该 worktree。
1. worktree 删除:`git_worktree_remove(repo, name)`;若该 worktree 有打开页签 → 返回需确认 → 前端二次确认 → 关页签后删除。

## 5. 技术要点与实现要领

- git2:`Repository::open`/`discover`(status/diff/branch 用 git2);worktree **统一走 shell `git worktree add/list/remove`**(git2 建新分支支持弱);自建 worktree 落 `.claude/worktrees/<name>`、分支 `worktree-<name>`(对齐 claude `--worktree`),并确保 `.claude/worktrees/` 入 `.gitignore`。
- 页签↔worktree 按目录绑定:面板记录其 cwd 所属 worktree;主工作区页签归”主”;一个页签对应一个 worktree,一个 worktree 可多页签。
- 操作页面 = Dockview 布局容器:切换 = 卸载当前布局 JSON、加载目标布局 JSON;终端会话不随切换销毁(后台保活,隐藏而非 dispose)。
- 重启不恢复页签:启动展示空白”打开终端”页;但**持久化项目列表 / 各项目操作页面结构(L3 名)/ 设置 / 键位**(见架构”配置持久化”),仅不恢复页签内容与 PTY 会话。

## 6. 注意事项与坑

- worktree 统一来源以 `git worktree list` 为准(claude 用 `--worktree` 建的也落 `.claude/worktrees/`、分支 `worktree-<name>`);后续 Phase 5 可经 `WorktreeCreate`/`WorktreeRemove` hook 实时感知 claude 的增删,免轮询。
- 删 worktree 前确认无未提交改动/无打开页签;破坏性操作必须二次确认。
- 切操作页面时保活的终端不能丢输出(后台仍接收并缓冲)。

## 7. 验收

### 7.1 自动化验收

- L1:`git_is_repo_detects`(仓库/非仓库);`worktree_list_add_remove`(对临时仓库,断言列表变化、目录创建/移除);`worktree_remove_blocked_when_open`(有页签时返回需确认信号)。
- L2:`add_project_appends_store`;`switch_page_swaps_layout`(布局 JSON 往返一致、会话保活);`new_worktree_opens_page_with_cwd`(新页面终端 cwd=worktree 路径,spy `pty_spawn` 参数)。

### 7.2 人工验收

- 侧栏三级树展开/交互正确;切操作页面整套布局切换且终端仍在跑;新建/删除 worktree 流程顺畅且确认到位。

## 8. 完成定义(DoD)

- L1/L2 全绿;人工清单勾选;不破坏回归集。
