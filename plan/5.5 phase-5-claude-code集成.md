# Phase 5 — Claude Code 集成(开发文档)

> 目标:claude 活动状态(页签状态点 + 操作页面节点聚合)、桌面通知、statusline、历史会话(–resume)。不把 claude 当黑盒——读其 hooks / transcript / statusline。

## 1. 目标与范围

- 交付:活动状态映射与展示、桌面通知(需介入/完成)、statusline 解析、历史会话列表与 `--resume`。
- 不在范围:图片粘贴/键位/性能(Phase 6)。

## 2. 前置依赖

- Phase 1(终端、sessions store)、Phase 2(操作页面节点)。

## 3. 涉及的模块与文件

- 后端:`claude/{mod.rs, hooks.rs(装 hook + 本地 HTTP 回传服务), transcript.rs, statusline.rs}`、`notify/mod.rs`。
- 前端:`features/notifications/`、`features/history/`、`stores/sessions.ts`(状态字段)、`ipc/claude.ts`、侧栏(操作页面节点聚合色)。

## 4. 开发内容(分任务)

1. hooks 安装 + 映射:向全局 `~/.claude/settings.json` 装 `type:http`、`async` hook(事件 POST 到本应用本地 HTTP 服务;127.0.0.1 + 随机端口 + token;header 带 `X-Pane-Id: $SLTERM_PANE_ID` 配 `allowedEnvVars`;以 `SLTERM_PANE_ID` 存在为门、模拟器外即 no-op;安装可逆);`hooks.rs` 起本地 HTTP 服务收事件 → **锁存状态机**映射:`UserPromptSubmit`/`PreToolUse`→运行中(锁存)、`Stop`→完成、`PermissionRequest`/`Elicitation`→需介入、`Notification(idle_prompt)`→等输入、`SubagentStop` **不**算完成;会话↔页签按回传 `pane_id` 精确关联(非猜 cwd)。
1. 状态展示:更新 `sessions` store → 终端页签状态点;操作页面节点显示聚合(需介入 > 运行中 > 空闲)。
1. 通知:`notify` 桌面 toast 仅”需介入/完成”且对应页签未聚焦时弹;点击 → 聚焦该页签。
1. statusline:仅**提供**一个 statusLine 脚本——claude 把会话 JSON 喂给它、它打印到 claude TUI 底部(`context_window.used_percentage` / `cost.total_cost_usd` / `rate_limits`);模拟器不解析、不另造组件。脚本兜底:首个 API 调用前与 `/compact` 后 `current_usage` 为 null、`used_percentage` 仅计输入 token。
1. 历史会话:`claude_sessions_list(project) -> [{id, title, time}]`(优先读全局索引 `~/.claude/history.jsonl` 按项目路径过滤,title 取首条 prompt 截断);点击 → `claude_resume` 生成 `claude --resume <id>`,**必须在该会话原 cwd**(项目/worktree 目录;`~/.claude/projects/<dashed>` 路径转义)运行,否则报 "No conversation found"。

## 5. 技术要点与实现要领

- 思考态(提交提示→首个工具)由 `UserPromptSubmit` 覆盖为”运行中”(无需再近似空闲)。
- 事件回传走 claude 原生 `type:http` hook → 本应用本地 HTTP 服务(`async` 不阻塞 claude,localhost 近即时);`X-Pane-Id` header 携带 `SLTERM_PANE_ID` 以正确归属页签。
- transcript 是 `~/.claude/projects/.../*.jsonl`;只读解析,勿写。
- 通知门控独立成函数 `should_notify(state, tab_focused)` 便于测试。

## 6. 注意事项与坑

- 多会话并行时事件要正确路由到对应页签(别串台)。
- 通知不要在聚焦时打扰;仅 completion/permission 弹,其余只标色。
- claude 是外部进程,CI 不跑真 claude → 用录制事件/transcript 样本;claude 未安装时给出优雅提示而非崩溃。

## 7. 验收

### 7.1 自动化验收

- L1/L2(基于样本):
  - `hook_event_maps_to_state`:`UserPromptSubmit`/`PreToolUse`→运行中、`PermissionRequest`/`Elicitation`→需介入、`idle_prompt`→等输入、`Stop`→完成、`SubagentStop` **不**翻完成,对样本逐一正确。
  - `pane_id_routing`:同 cwd 多会话事件按 `pane_id` 不串台。
  - `page_node_aggregates_priority`:页面节点 = 子项最高优先级。
  - `should_notify_only_when_unfocused`:门控真值表。
  - `statusline_script_outputs`:statusLine 脚本对样本 JSON 输出含 context%/cost,并兜底 `current_usage` 为 null。
  - `history_list_and_resume_command`:`history.jsonl` 样本 → 列表 + 正确 `--resume <id>` 命令与 cwd。

### 7.2 人工验收

- 真跑 claude:触发授权 + 完成 → 状态色正确、toast 实际弹出(仅未聚焦)、点击跳转;statusline 显示实时数字;resume 真续上。
