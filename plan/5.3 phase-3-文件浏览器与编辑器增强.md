# Phase 3 — 文件浏览器与编辑器增强(开发文档)

> 目标:次级侧栏文件浏览器(git 着色、双击打开)、CM6 语法高亮、行内 git diff 边栏、查找、外部改动处理。

## 1. 目标与范围

- 交付:文件浏览器 + 语法高亮 + 行内 diff 边栏 + 文件名 git 着色 + 查找(Ctrl+F)+ 外部改动处理。
- 不在范围:状态栏、claude。

## 2. 前置依赖

- Phase 1(编辑器面板、Dockview)、Phase 2(工作区选择、git 基础)。

## 3. 涉及的模块与文件

- 后端:`fs/{mod.rs, watch.rs}`、`git/mod.rs`(status/diff)。
- 前端:`features/explorer/`、`panels/editor/{EditorPanel.tsx, useCodeMirror.ts, gitGutter.ts}`、`theme/colors.ts`、`ipc/{fs.ts, git.ts}`。

## 4. 开发内容(分任务)

1. 文件浏览器:`fs_read_dir(path) -> [entry]`;树展示当前选中工作区根;双击 → 在焦点操作页面开 editor 面板;隐藏 `.claude/worktrees/`。
1. 文件监听:`fs_watch(path)`(notify)→ `FsEvent` → 刷新浏览器与 git 状态。
1. 语法高亮:CM6 按扩展名加载语言包。
1. 行内 diff 边栏:`git_diff(path) -> hunks`;`gitGutter.ts` 把增/改/删行映射到 JetBrains 暗色行色;基线 HEAD。
1. 文件名 git 着色:`git_file_status(repo) -> {path: status}`;浏览器用 JetBrains 暗色文件色。
1. 查找:CM6 `@codemirror/search`(Ctrl+F)。
1. 外部改动:监听到已打开文件被改 → 干净则自动重载、脏则弹窗(保留我的/重载/看差异——"看差异"是未保存内容 vs 磁盘新版,与 HEAD 行内 diff 不同视图;弹窗暂未就绪时先只给"保留我的/重载")。

## 5. 技术要点与实现要领

- 配色 token 全在 `theme/colors.ts`:
  - 文件名色:Modified `#6897BB`、Added `#629755`、Untracked `#D1675A`、Deleted `#6C6C6C`、Renamed `#3A8484`、Conflict `#D5756C`、Ignored `#848504`。
  - 行内 diff 色:修改 `#374752`、新增 `#384C38`、删除 `#656E76`、仅空白 `#4C4638`。
  - 组件只引 token,禁止硬编码颜色(架构约束第 6 条)。
- CM6 diff 边栏用 `gutter()` + `GutterMarker`;git hunks → 行号区间 → marker 颜色。
- notify 加去抖,避免保存触发的抖动风暴。

## 6. 注意事项与坑

- 编辑器未保存内容 claude 看不到(它读盘上版本)。
- 外部改动判定分”干净/脏”两支;脏时绝不静默覆盖用户编辑。
- 大目录树懒加载,别一次读满。

## 7. 验收

### 7.1 自动化验收

- L1:`git_status_diff_file_status`(对临时仓库正确);`fs_read_dir_filters_worktrees`(过滤 `.claude/worktrees/`);`fs_watch_fires_on_change`。
- L2/L3:`gutter_maps_hunks_to_tokens`(diff hunk → 正确行色 token,快照);`filename_status_color_token`(各状态 → 正确 token,把”肉眼比色”变成断言);`external_change_clean_reloads` / `external_change_dirty_prompts`(模拟 FsEvent + mock store 分支)。

### 7.2 人工验收

- 配色与 JetBrains 肉眼一致;diff 边栏观感正确;双击打开、查找、重载/弹窗体验正常。

## 8. 完成定义(DoD)

- L1/L2/L3 全绿;人工(主要是配色与观感)勾选;不破坏回归集。
