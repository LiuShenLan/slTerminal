# Phase 6 — 体验完善(开发文档)

> 目标:图片粘贴、自定义键位、暗色主题/字体可配置、性能打磨。收尾阶段。

## 1. 目标与范围

- 交付:图片粘贴(临时文件 + 路径)、自定义键位(仅 UI 动作)、暗色主题/字体可配置、性能打磨(合批/WebGL/大输出)。
- 不在范围:无(收尾)。

## 2. 前置依赖

- Phase 1–5。

## 3. 涉及的模块与文件

- 后端:剪贴板/临时文件(可放 `fs/` 或新增 `clipboard` util)。
- 前端:`stores/settings.ts`、键位分发层(`lib/` 或 `features/`)、`theme/`、`panels/terminal/`(性能)。

## 4. 开发内容(分任务)

1. 图片粘贴:`clipboard_image_to_temp() -> {path}`(剪贴板有图 → 存临时 PNG → 返回路径);粘贴时把路径文本经 `pty_write` 写入终端(claude 读其为图片引用)。
1. 自定义键位:`settings` 维护”UI 动作 ↔ 键”映射(新建/分屏/切换/复制等);非 UI 键透传 claude。
1. 主题/字体:`settings` 暴露暗色主题变量与字体;`theme/` 应用。
1. 性能:确认合批渲染、WebGL 生效;大输出不卡;必要时后端攒批。

## 5. 技术要点与实现要领

- 剪贴板取图用 arboard / Tauri clipboard;Windows 上以”临时文件 + 路径”喂 claude 最稳。
- 键位分发表集中:UI 动作走表、其余 `pty_write` 透传;复制固定 Ctrl+Shift+C。
- WebGL 驱动异常可切 canvas(一个开关)。

## 6. 注意事项与坑

- 别拦截要透传给 claude 的键(尤其 claude 自己的快捷键)。
- 临时图片要清理(会话结束/定期)。

## 7. 验收

### 7.1 自动化验收

- L1:`clipboard_image_saved_to_temp`(mock 剪贴板 → 文件存在、是 PNG)。
- L2:`paste_inserts_path`(粘贴图 → 经 `pty_write` 写入返回路径);`keybinding_dispatch`(动作表分发正确、未映射键透传)。
- L3/性能:`large_output_no_hang`(灌大输出突发 → 处理耗时阈值内、不挂);`batching_coalesces`(合批单测)。

### 7.2 人工验收

- 真截图粘进 claude 并确认收到;自定义键位手感;大 `cat` 不卡;字体/主题切换生效。

## 8. 完成定义(DoD)

- L1/L2/L3 全绿;人工勾选;不破坏回归集。
