# Phase 4 — git / Python 状态栏(开发文档)

> 目标:窗格下方状态栏——git 分支、diff/dirty 概要、Python 虚拟环境。

## 1. 目标与范围

- 交付:状态栏展示分支、diff/dirty 概要、Python 虚拟环境,随当前工作区/终端 cwd 刷新。
- 不在范围:claude。

## 2. 前置依赖

- Phase 2/3(git 能力、fs_watch、工作区上下文)。

## 3. 涉及的模块与文件

- 后端:`git/mod.rs`(分支/概要);venv 由 `pty/` 的 OSC 序列解析(Phase 1 已埋点),状态栏只消费。
- 前端:`features/statusbar/`、`stores/`(当前工作区上下文)、`ipc/git.ts`。

## 4. 开发内容(分任务)

1. 分支:`git_branch(repo) -> {name, ahead, behind}`(或并入 status)。
1. diff/dirty 概要:`git_diff_summary(repo) -> {added, modified, deleted, staged}`。
1. Python 环境:`python_env_detect(osc_payload) -> {kind, name}`(从 profile 回传的 OSC venv 序列解析当前激活:.venv / virtualenv / conda / 无)。
1. 状态栏:随当前聚焦终端的 cwd/工作区刷新(订阅 store + fs_watch)。

## 5. 技术要点与实现要领

- 分支/概要走 git2;ahead/behind 用 graph ahead-behind。
- venv 探测:显示**当前激活**环境,经 Phase 1 profile 的自定义 OSC 序列回传 `$env:VIRTUAL_ENV` / `$env:CONDA_DEFAULT_ENV` / conda 标记(宿主读不到子 shell env,不能只查目录 `.venv/pyvenv.cfg`——那只反映存在性、非激活);只展示,不自动激活。
- cwd 来源:优先终端 OSC 7(**Phase 1 profile 已提供**,非 Phase 5),否则按当前工作区根。
- 性能:大仓库 status 慢 → 后端缓存 + 去抖刷新,别每次输出都全量算。

## 6. 注意事项与坑

- 非 git 目录 / 无 venv 时状态栏优雅留空。
- 频繁 fs 事件要去抖,避免状态栏抖动与 CPU 飙升。

## 7. 验收

### 7.1 自动化验收

- L1:`branch_ahead_behind`、`diff_summary_counts`、`python_env_detect`(对构造的 OSC venv 序列样本,解析出激活环境名)。
- L2:`statusbar_renders_branch_diff_env`(store → 状态栏文本/图标正确)。

### 7.2 人工验收

- 真实仓库/venv 下状态栏与实际一致;切换 cwd/工作区时刷新正确。
