# Phase 1 — 核心外壳(开发文档)

> 目标:统一分屏页签区里能开 PowerShell、能开文件编辑、能拖拽分屏调比例、能跑 claude。这是最硬的核(PTY + 渲染 + 分屏页签)。验收方法学见《完整开发计划》。

## 1. 目标与范围

- 交付 M1 全部:1.1 PowerShell 终端面板;1.2 打开并编辑文件;1.3 多页签管理;1.4 拖拽分屏/嵌套/调比例;1.5 跑 claude。
- 不在范围:侧边栏/项目结构、文件浏览器、git/状态栏、claude 状态集成(后续 phase)。

## 2. 前置依赖

- Phase 0(骨架 + 测试基建)。

## 3. 涉及的模块与文件

- 后端:`pty/{mod.rs, session.rs, spawn.rs, io_loop.rs, shell.rs}`、`state.rs`(挂 PtyState)、`fs/mod.rs`(读写文件)、`lib.rs`(注册命令)、`capabilities/default.json`(放行)。
- 前端:`ipc/{pty.ts, fs.ts, client.ts}`、`panels/terminal/{TerminalPanel.tsx, useXterm.ts, theme.ts}`、`panels/editor/{EditorPanel.tsx, useCodeMirror.ts}`、`workspace/{Workspace.tsx, panelRegistry.ts, layoutSerde.ts}`、`stores/sessions.ts`(会话 id↔面板)。

## 4. 开发内容(分任务)

### 1.1 PowerShell 终端面板

- 后端命令:
  - `pty_spawn(shell, cwd, cols, rows, on_output: Channel<PtyEvent>) -> { id }`
  - `pty_write(id, data)` / `pty_resize(id, cols, rows)` / `pty_kill(id)`
  - `PtyEvent`:`Output { bytes }` | `Exit { code }`。
  - `Session`:持 `PtyPair`、master writer、child、reader 线程;`PtyState = RwLock<HashMap<String, Session>>` 挂 `AppState`;`spawn.rs` 内 `SPAWN_LOCK: Mutex<()>`。
- **profile 注入 + 关联埋点**(本 phase 交付):`shell.rs` 以 `-File <profile.ps1>` 启动 PowerShell,profile 随 prompt 发 OSC 7(cwd)/ OSC 133(命令边界 + 退出码)/ 自定义 venv 序列;spawn 经 `CommandBuilder.env` 注入唯一 `SLTERM_PANE_ID`(供 Phase 5 hook 关联)。
- 前端:`useXterm.ts` 建 `Terminal` + `WebglAddon` + `FitAddon`;`TerminalPanel` 挂载即 `ipc.pty.spawn`,订阅 Channel → `term.write`,`term.onData` → `ipc.pty.write`,`ResizeObserver` → fit → `ipc.pty.resize`;在 `panelRegistry` 注册 `terminal`。

### 1.2 打开并编辑文件

- 后端:`fs_read_file(path) -> String`、`fs_write_file(path, content)`。
- 前端:`EditorPanel` 用 CM6(`useCodeMirror.ts`);打开即 read,Ctrl+S → write;注册 `editor` 面板。

### 1.3 多页签管理

- `Workspace` 用 `DockviewReact`;新建终端/文件 = `api.addPanel`(加到焦点分组之后);关最后一个 → 显示”打开终端”空白态(Dockview watermark/fallback)。

### 1.4 拖拽分屏 + 嵌套 + 调比例

- 启用 Dockview 内置 DnD/splitview;`layoutSerde.ts` 封装 `api.toJSON/fromJSON`;`onDidLayoutChange` 把当前操作页面布局存到 `stores/layout`。

### 1.5 跑 claude

- 用 `attachCustomKeyEventHandler` 正确处理 Shift+Tab、Ctrl 组合等并送 `pty_write`;验证备用屏/真彩渲染。

## 5. 技术要点与实现要领

- portable-pty:`native_pty_system().openpty(PtySize{rows,cols,..})` → `slave.spawn_command(CommandBuilder)`;master 拆 reader/writer;reader 在独立线程 `read` → 经 Channel 发 `Output`;`resize` 调 master。
- ConPTY:spawn 全程握 `SPAWN_LOCK`;cwd 传入前 `replace('/', "\\")`;shell 选择 `pwsh.exe`→`powershell.exe`→`cmd.exe`(`shell.rs`)。
- 渲染合批:Channel 输出在前端按帧(rAF≈16ms)或字节阈值合并再 `term.write`(或后端攒批),减少写次数,防 `cat 大文件` 卡死。
- 每个 `xterm.Terminal` 随面板存活;切走只隐藏 DOM,不 dispose、不 kill PTY。
- Dockview:`components` = panelRegistry;`addPanel({id, component, params})`;空态用 watermark。

## 6. 注意事项与坑

- camelCase(JS)↔ snake_case(Rust)参数名。
- `pty_kill` 必须回收 reader 线程与子进程,避免孤儿(L5 验)。
- DnD 落点用”悬停分组 + 靠近边”,是 Dockview 内置行为,别自造。
- CM6 别中途换 Monaco(Electron 假设坑)。

## 7. 验收

### 7.1 自动化验收

- L1(`cargo test`):
  - `pty_roundtrip`:spawn pwsh → `pty_write("echo hello\r\n")` → 收到的 `Output` 拼接含 `hello`。
  - `pty_resize_applies`:resize 后跑依赖宽度的命令,输出按新宽度换行。
  - `pty_kill_no_orphan`:kill 后会话从 PtyState 移除,子进程退出(查进程/日志)。
  - `fs_read_write_roundtrip`:写临时文件→读回一致。
  - `osc_cwd_venv_parsed`:注入 profile 后从输出解析出 OSC 7 的 cwd 与自定义 venv 序列(对构造样本)。
- L2(Vitest + mockIPC):
  - `terminal_panel_spawns_on_mount`:挂载调用 `pty_spawn`(spy)。
  - `terminal_input_writes`:模拟 `term.onData` → `pty_write` 参数正确。
  - `editor_saves_edited_content`:编辑后 Ctrl+S → `fs_write_file` 带编辑后文本。
  - `closing_last_panel_shows_empty_state`:关掉最后面板渲染空白态。
- L3(`@xterm/headless`;Termless 0.x 可选):
  - `renders_pwsh_prompt`:feed 录制的 pwsh 输出 → `term.screen` 含提示符。
  - `claude_tui_snapshot`:feed 录制的 claude 输出 → 缓冲/单元格/颜色匹配黄金快照。
  - `key_encoding_shift_tab` / `key_encoding_ctrl_c`:按键产生正确字节序列。
- L4(可选,关键流程):启动应用 → 点”打开终端” → 输入 `echo e2e` → DOM 出现 `e2e`。
