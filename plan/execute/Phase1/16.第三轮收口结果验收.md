## 三、已知注意事项

1. **L4 teardown 噪声**：`Failed to clear mock store` 为 `@wdio/tauri-service` session 销毁时序问题，不影响 spec 结果。L4 启动前的 `Failed to get window states` 同理。
2. **`libpng warning: iCCP`**：WebView2 内嵌渲染管线，非我方代码。
3. **Ctrl+W/T/N**：WebView2 硬编码关闭，不可通过 prevent-default 配置。
4. **L4 E2E 文本验证**：通过 `__e2e_writeToTerminal()` 直接写入标记文本验证缓冲机制。PTY 输出因 embedded mode 下 PowerShell 初始化延迟可能未在 timeout 内到达——L1 集成测试（`pty_roundtrip`）已覆盖 PTY 端到端输出。
5. **G1a `useRef` 守卫**：在 `tauri build`（Vite 生产模式）下 React StrictMode 不双调 `useEffect`，守卫为防御性设计。
6. **G1b `spawn_blocking`**：Tokio 阻塞线程池上限 512，slTerminal ~10 终端远未触及。`ClosePseudoConsole` 在 pre-Win11 24H2 (build < 26100) 上永久阻塞，`spawn_blocking` 将其从 IPC worker 移走——IPC 不阻塞。
7. **G2 五层守卫**：`new Terminal({ cols: 80, rows: 24 })` 安全网 + RO 门控（rAF 去重）+ `canFit()` 显式守卫 + setTimeout 三级回退 + `isDisposedRef` 防止销毁后操作。WebView2 ResizeObserver 可能触发 {0,0} 后静默（CSSWG #7808）——Layer 5 setTimeout 回退覆盖此场景。
8. **G2 `contain: strict` CSS**：有意裁剪——单终端布局不需要，会破坏 `position: fixed`。Daintree 项目自身在 grid cell 中禁用（#4713）。
9. **F3 掩码 `& 0x0FFFFFFF`**：RtlGetNtVersionNumbers 返回 raw build 的高 4 位为零售版标志（0xF）。`& 0x0FFFFFFF` 清除标志位保留低 28 位，语义清晰于旧掩码 `& 0xFFFF`。当前所有 Windows build < 65535，两种掩码功能等价。
10. **F3 `windowsPty` 生效路径**：独立 useEffect 监听 `windowsBuildNumber` 异步更新——不与主 useEffect 的 StrictMode 守卫冲突。xterm.js `_handleWindowsPtyOptionChange` 在 Terminal 创建后设置新对象仍会触发。
11. **D3 CodeMirror Compartment**：`Compartment.reconfigure()` 不丢失文档状态（`EditorState.doc` 保留）。编辑器场景文件通常 < 100KB，parser 重解析开销可忽略。
12. **D4 `src/ipc/dialog.ts`**：薄 re-export，`save()` 本身就调用 `invoke()`——走相同 IPC 通道，零额外安全风险。聚合到 `src/ipc/` 满足架构约束 #1 的集中式审计要求。
13. **G4 无副作用**：slTerminal 当前无文件拖放功能（无 `onDragDropEvent` 监听），`dragDropEnabled: false` 仅恢复 HTML5 DnD 通路。
14. **G3 已知风险 #2268**：`@tauri-apps/plugin-dialog` import 即可能导致 `<input>` 失焦——slTerminal 中 dialog 仅在 `useCodeMirror.ts` 使用（CodeMirror 不含 `<input>`），实际影响极小。
