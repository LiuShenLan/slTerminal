E2E 流程：
1. `waitUntil` 等待 Dockview API → `window.__dockviewApi` 就绪
2. `execute` 调用 `addPanel({ component: 'terminal', renderer: 'always' })` 创建终端
3. `waitUntil` 等待 `__e2e_sessionReady === true`（PTY spawn 成功）
4. `execute` 调用 `__e2e_writeToPty('echo e2e_marker\r\n')` 写入命令
5. `execute` 调用 `__e2e_writeToTerminal('e2e_marker_direct\r\n')` 直接写入缓冲
6. `waitUntil` 轮询 `__e2e_getTerminalText()` → 断言含 `e2e_marker`

## 三、已知注意事项

1. **L4 teardown 噪声**：`Failed to clear mock store` 为 `@wdio/tauri-service` session 销毁时序问题，不影响 spec 结果。
2. **L4 startup 噪声**：`Failed to get window states: Tauri core.invoke not available after 5s timeout` 为 embedded driver 初始化时序，`core.invoke` 就绪前查询超时，不影响 spec。
3. **`libpng warning: iCCP`** 来自 WebView2 内嵌渲染管线，非我方代码。
4. **Ctrl+W/T/N** 在 WebView2 中硬编码关闭，不可通过 prevent-default 配置开关——符合预期。
5. **L4 E2E 文本验证**：当前通过 `__e2e_writeToTerminal()` 直接写入标记文本验证缓冲机制。PTY 输出因 embedded mode 下 PowerShell 初始化延迟可能未在 timeout 内到达——此限制不影响实际功能，L1 集成测试中 PTY 输出已验证。
