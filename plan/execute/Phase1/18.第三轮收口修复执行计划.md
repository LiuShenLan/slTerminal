## 3. 修复方案

### 3.1 前端核心：`useXterm.ts` 重构（方案 B——termoras rAF 轮询）

**文件**：`src/panels/terminal/useXterm.ts`

**变更清单**：

1. **主 effect 重写**：`term.open()` → rAF 轮询 `offsetWidth>0`（最多 30 帧/~500ms）→ `fitAddon.fit()` → 获取真实 cols/rows → `pty.spawn()`。超时回退 80×24 spawn
2. **RO 移回主 effect 内部**（随 Terminal 重建自动重建——解决 F2）
3. **`windowsBuildNumber` 从主 effect deps 移除**（解决 F3）——F3 独立 effect 保留（动态更新 `windowsPty`）
4. **`isDisposedRef.current = false` 在 effect body 首行复位**（防御性，解决 F1）
5. **`fonts.ready` fit 外包裹 rAF**（防御 P1）
6. **PTY resize 加 100ms debounce**（Tabby 模式）
7. **删 G2 独立 RO effect**（L221–271）——RO 已移回主 effect
8. **`canFit()` 保留但条件 5（isDisposedRef）在 rAF 轮询路径上不使用**——轮询自身检查 `offsetWidth>0` 和 `terminal.element`

**简化后的 `canFit()` 调用链**：

```typescript
// rAF 轮询不再走 canFit——直接检查 offsetWidth>0 和 terminal.element
const tryConnect = () => {
    const el = container;
    if (!el || el.offsetWidth === 0 || el.offsetHeight === 0) {
        attempts++;
        if (attempts < 30) requestAnimationFrame(tryConnect);
        else {
            // 超时回退：80×24 spawn
            pty.spawn({ panelId, cols: 80, rows: 24 }, handlePtyOutput)...;
        }
        return;
    }
    fitAddon.fit();
    const dims = fitAddon.proposeDimensions();
    const c = dims?.cols ?? 80;
    const r = dims?.rows ?? 24;
    pty.spawn({ panelId, cols: c, rows: r }, handlePtyOutput)...;
};

// fonts.ready 中的 fit 仍走 canFit（防御性）
document.fonts.ready.then(() => {
    if (canFit(term, fitAddon, container, isDisposedRef)) {
        fitAddon.fit();
    }
});

// RO doFit 仍走 canFit（防御后续 resize）
const doFit = () => {
    if (!canFit(term, fitAddon, container, isDisposedRef)) return;
    fitAddon.fit();
    const dims = fitAddon.proposeDimensions();
    if (dims && sessionIdRef.current) {
        ptyResizeTimer = setTimeout(() => {
            pty.resize(sessionIdRef.current!, dims.cols, dims.rows);
        }, 100); // 100ms debounce
    }
};
```

## 7. 注意事项

1. **`canFit()` 保留不删**——被 L2 单元测试依赖（`canFit.test.ts`），且 fonts.ready + RO doFit 仍用它做防御性守卫
2. **F3 effect 保留**——`windowsBuildNumber` 异步更新 `windowsPty` 选项，功能正确；只是之前被主 effect 覆盖
3. **`isDisposedRef` 保留**——用于 cleanup 时阻止异步回调操作已销毁终端
4. **PTY resize debounce 100ms**——每 100ms 最多发一次 resize 到 ConPTY，防止拖拽窗口时 60 次/秒轰炸
5. **rAF 轮询上限 30 帧**——~500ms。正常情况 P4 CSS 修复后 0-1 帧就绪；dockview overlay 最多 2 帧；30 帧充裕
6. **超时回退 80×24**——极端情况（容器永不就绪）降级但不丢功能。后续 RO doFit 会纠正尺寸
7. **不修改 `TerminalPanel.tsx`**——硬编码 `cols:80, rows:24` 保留（仅作 fallback），实际 PTY spawn 用 fit 后尺寸
8. **不修改 `canFit()` 导出签名**——测试文件依赖此导出
9. **`App.css`**——项目全局样式文件，所有 Dockview 面板自动受益
