## 3. 人工验证失败项分析

### 3.1 关闭面板卡死（阻断）

**现象**（报告 §2.1 #2, #8）：关闭全部面板后应用显示"（未响应）"；点击"关闭其他"和"全部关闭"同样卡死。

**根因分析**（网络检索 + 代码审查）：

1. **Dockview `onReady` 中同步 `addPanel` 的已知竞态**（dockview-core #341 / #866）：React Strict Mode 下 `onReady` 触发两次，`addPanel` 在内部视口未完全初始化时操作可能损坏状态。当前代码在 `onReady` 中同步调用 `api.addPanel()`——这正是已知问题模式。修复方案：将 api 存入 state，通过 `useEffect` 延迟调用 `addPanel`。

2. **Tauri 2 + WebView2 事件循环死锁**（tauri #14088 / #11254 / #13498）：当所有面板关闭触发级联 cleanup（`pty.kill()` → `child.kill()` + `reader_handle.join()`），多个同步 I/O 在 UI 线程 WebView2 事件循环中同时执行可能死锁。"关闭所有面板→显示 Watermark"的过渡与 Tauri 的窗口事件处理存在已知交互问题。

3. **Watermark 状态机异常**：`closeAll` 后 Dockview 进入 Watermark 状态。若 `IWatermarkPanelProps.containerApi` 在面板全关后处于过渡态，可能触发渲染循环。

**建议**：① `onReady` 中 `addPanel` 改用 `useState` + `useEffect` 延迟调用（deckview-core ≥4.2.2 修复方案）② 关闭所有面板时避免同步 kill 多个 PTY ③ 始终保留至少一个面板避免进入 Watermark 过渡态

### 3.2 终端工具栏"+"创建后不可用（阻断）

**现象**（报告 §2.1 #5）：工具栏"+"新建终端面板成功，但终端无法使用（截图证实）。

**根因分析**：与 §3.1 根因①高度相关——`onReady` 中的同步 `addPanel` 已触发资源竞态，后续通过 `RightHeaderActions` → `containerApi.addPanel()` 创建的终端面板可能继承了已损坏的 Dockview 内部状态。

**补充分析**：`RightHeaderActions` 中 `addPanel` 的 `id` 使用 `terminal-${Date.now()}` 生成，与 `onReady` 创建的 `terminal-init-${Date.now()}` 无冲突。但 Dockview 内部 panel map 若因 onReady 竞态而损坏，则后续所有面板的 PTY spawn 都可能受影响。

**建议**：优先修复 §3.1 根因①（onReady 竞态），该问题修复后终端创建可能自然恢复。

### 3.3 Ctrl+S 无反应（中等）

**根因明确**（代码审查）：`useCodeMirror.ts:28-37` 的 `handleSave` 检查 `if (!view || !path) return;`。所有面板创建入口（右键菜单、Watermark 按钮）均未传入 `filePath` 参数 → `path` 为 `undefined` → `handleSave` 静默返回。

```typescript
// Workspace.tsx 右键菜单——缺 filePath
api.addPanel({
  id: newEditorId,
  component: "editor",
  params: { panelId: newEditorId },
  // 缺少 filePath 参数！
});
```

**修复**：① 编辑器面板创建时支持可选 `filePath` 参数（或弹出文件选择器）② 若无 `filePath`，提供"另存为"对话框。这是执行计划 Agent A7 的残留缺口——编辑器 Ctrl+S 保存从未在用户交互层面验证过。

### 3.4 页签无法拖拽（中等）

**现象**（报告 §2.6 #25）：拖拽 tab 到窗口边缘无分屏反应。

**分析**：Dockview 的拖拽分屏是内置行为，应默认启用。可能原因：① 当前只有单个面板时拖拽行为受限 ② Dockview 的 `dragEnabled` / `disableDrag` 选项。

**建议**：检查 `DockviewReact` 的 props 中是否有禁用拖拽的配置。

### 3.5 编辑器创建无 `filePath` 参数

与 §3.3 同根因。所有编辑器入口均不传 `filePath`，导致编辑器仅可作为"空白记事本"使用，无法执行保存操作。
