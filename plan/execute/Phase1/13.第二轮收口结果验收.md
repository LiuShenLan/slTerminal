## 三、已知注意事项

1. **L4 teardown 噪声**：`Failed to clear mock store` 为 `@wdio/tauri-service` session 销毁时序问题，不影响 spec 结果。
2. **L4 startup 噪声**：`Failed to get window states: Tauri core.invoke not available after 5s timeout` 为 embedded driver 初始化时序，`core.invoke` 就绪前查询超时，不影响 spec。
3. **`libpng warning: iCCP`** 来自 WebView2 内嵌渲染管线，非我方代码。
4. **Ctrl+W/T/N** 在 WebView2 中硬编码关闭，不可通过 prevent-default 配置开关——符合预期。
5. **L4 E2E 文本验证**：通过 `__e2e_writeToTerminal()` 直接写入标记文本验证缓冲机制。PTY 输出因 embedded mode 下 PowerShell 初始化延迟可能未在 timeout 内到达——L1 集成测试（`pty_roundtrip`）已覆盖 PTY 端到端输出。
6. **G1a `useRef` 守卫**：在 `tauri build`（Vite 生产模式）下 React StrictMode 不双调 `useEffect`，守卫为防御性设计——开发模式下也正常工作。`isReadyRef` 初始化器在 mount/remount 时只运行一次，remount 后 `isReadyRef.current` 保持 `true`，跳过第二次 `onReady`。
7. **G1b `spawn_blocking` 线程池**：Tokio 默认阻塞线程池上限 512。slTerminal 终端量级（~10）远未触及。`ClosePseudoConsole` 在 pre-Win11 24H2 (build < 26100) 上永久阻塞，`spawn_blocking` 将其从 IPC worker 移到独立线程——IPC 不阻塞（目标达成）。
8. **G3 `save()` 为 async IPC**：对话框期间 JS 事件循环继续运行，不阻塞 UI。
9. **G3 已知风险 #2268**：`@tauri-apps/plugin-dialog` import 即可能导致 `<input>` 失焦——slTerminal 中 dialog 仅在 `useCodeMirror.ts` 使用（CodeMirror 不含 `<input>`），实际影响极小。
10. **G4 无副作用**：slTerminal 当前无文件拖放功能（无 `onDragDropEvent` 监听），`dragDropEnabled: false` 仅恢复 HTML5 DnD 通路，不影响现有功能。
