## 5. Windows 边缘案例分析

### 5.1 Windows 进程终止事件可靠性矩阵

| 终止方式 | `CloseRequested`(JS) | `ExitRequested` | `Exit` | Drop 运行 | 文件损坏 |
|---------|:---:|:---:|:---:|:---:|:---:|
| 用户点 X | ✅ | ✅ | ✅ | ✅ | 无 |
| `taskkill`（无 `/f`） | ✅ | ✅ | ✅ | ✅ | 低 |
| **`taskkill /f`** | ❌ | ❌ | ❌ | ❌ | **HIGH** |
| 任务管理器结束任务（无响应） | ❌ | ❌ | ❌ | ❌ | **HIGH** |
| **系统关机/重启** | 可能 | 可能 | 可能 | 取决于时序 | **中-高** |
| `TerminateProcess` | ❌ | ❌ | ❌ | ❌ | **HIGH** |

**核心结论**：不能依赖任何退出事件保存数据。`TerminateProcess` 立即挂起所有线程，不执行任何用户态代码——无 `DllMain`、无 `atexit`、无 Rust `Drop`（Raymond Chen 官方博客确认）。

### 5.2 其他 Windows 风险

| 风险 | 等级 | 说明 | 来源 |
|------|------|------|------|
| Defender AV 拦截 `MoveFileExW` | HIGH（企业环境） | 每 50-100 次写入失败一次，返回 `ERROR_ACCESS_DENIED(5)` | dotnet/runtime#114230 |
| 多实例状态文件竞态 | HIGH | 两实例同时 write→逻辑丢失更新 | VS Code #134909 |
| `core.longpaths` 未启用 | HIGH | Win10 1607–Win11 2025 均默认禁用；git worktree 路径易超 260 字符 | Microsoft Q&A |
| 快速启动不保证刷盘 | HIGH | WriteFile→OS 缓存→休眠前 I/O 管理器尝试刷但不保证 | MSDN |
| CFA 受控文件夹访问 | LOW（默认） | `%USERPROFILE%\.slterminal\` 不在默认保护列表 | Microsoft Defender Docs |
| `dirs::home_dir()` 返回 UNC | MEDIUM（企业） | 全 profile 托管环境可能返回 `\\server\profiles$\user` | Super User |
