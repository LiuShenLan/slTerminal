# 工作树安全删除序列 — 完整规格

> 基于 slTerminal 现有代码库（`src-tauri/src/git/worktree.rs`、`src/stores/projects.ts`、`src/features/worktree/`），定义 worktree 安全删除的完整开发规格，覆盖预检→删除→清理→错误分类→测试五阶段。

---

## 1. 预检阶段 (Pre-check Phase)

预检在弹出确认对话框**之前**完成。所有检查结果汇总为一份 `PreCheckResult`，前端据此决定显示什么信息。

### 1.1 主工作树检测

**规则**：禁止删除主工作树（main worktree）。

**实现**：有三种检测方式，按可靠性排序：

**方式 A（最可靠，推荐）**：`git2::Worktree::name()` 对主工作树返回 `None`，对链接工作树返回 `Some("名称")`。

```rust
use git2::Repository;

fn is_main_worktree(repo_path: &str) -> Result<bool, git2::Error> {
    let repo = Repository::open(repo_path)?;
    for wt_name in repo.worktrees()?.iter() {
        let wt = repo.find_worktree(wt_name)?;
        if wt.name().is_none() {
            // 主工作树 — name() 返回 None
            return Ok(true);
        }
    }
    Ok(false)
}
```

**方式 B（备选）**：规范化路径后与 `Repository::workdir()` 比较。

```rust
let worktree_canon = Path::new(worktree_path).canonicalize()?;
let main_canon = repo.workdir().unwrap().canonicalize()?;
let is_main = worktree_canon == main_canon;
```

**方式 C（当前实现，有缺陷）**：解析 `git worktree list --porcelain` 输出，根据 `branch` 字段是否为 `main`/`master` 判断。缺陷：detached HEAD 时分支名为空，且分支名 `main` 不保证是主工作树（链接工作树也可以 checkout 到 main 分支）。

**建议**：前端 `WorktreeInfo.is_main` 字段由后端在 `git_worktree_list` 中通过**方式 B**（规范化路径比较）或**方式 A**（git2 `Worktree::name()`) 填充，替换当前基于分支名的判断。

**阻塞行为**：主工作树直接不显示删除按钮。前端在渲染 worktree 列表时，对 `isMain === true` 的条目禁用删除操作。

```typescript
const canDelete = !worktree.isMain;
```

### 1.2 未提交更改检测 (Dirty Check)

**规则**：worktree 有未提交更改时，应警告用户但允许强制删除。分三级：

| 级别 | 条件 | 处理 |
|------|------|------|
| Clean | 无任何更改 | 直接允许删除 |
| Dirty (可恢复) | 仅有 unstaged/untracked 更改 | 警告"有未保存更改，删除后无法恢复"，checkbox gate |
| Blocked | 有进行中操作(merge/rebase/cherry-pick) | 阻止删除，提示"请先完成或中止当前操作" |

**实现**（新增 `git_worktree_status` 命令）：

```rust
// src-tauri/src/git/worktree.rs

/// worktree 状态 DTO
#[derive(Debug, Clone, Serialize, Deserialize, Default)]
#[serde(rename_all = "camelCase")]
pub struct WorktreeStatusDto {
    pub is_dirty: bool,
    pub has_staged: bool,
    pub has_unstaged: bool,
    pub has_untracked: bool,
    pub has_conflicts: bool,
    /// 进行中操作: "merge" | "rebase" | "cherry-pick" | "revert" | "bisect" | null
    pub mid_operation: Option<String>,
    pub changed_count: usize,
    pub untracked_count: usize,
}

/// 获取 worktree 的 git 状态，用于删除前预检
#[tauri::command]
pub fn git_worktree_status(worktree_path: String) -> Result<WorktreeStatusDto, AppError> {
    let repo = git2::Repository::open(&worktree_path)
        .map_err(|e| AppError::Git(format!("无法打开仓库: {e}")))?;

    let mut dto = WorktreeStatusDto::default();

    // 1. 检测进行中操作（repo.state()）
    use git2::RepositoryState;
    dto.mid_operation = match repo.state() {
        RepositoryState::Clean => None,
        RepositoryState::Merge => Some("merge".into()),
        RepositoryState::Rebase
        | RepositoryState::RebaseInteractive
        | RepositoryState::RebaseMerge => Some("rebase".into()),
        RepositoryState::CherryPick
        | RepositoryState::CherryPickSequence => Some("cherry-pick".into()),
        RepositoryState::Revert
        | RepositoryState::RevertSequence => Some("revert".into()),
        RepositoryState::Bisect => Some("bisect".into()),
        _ => None,
    };

    // 2. 获取文件状态
    let mut opts = git2::StatusOptions::new();
    opts.show(git2::StatusShow::IndexAndWorkdir);
    opts.include_untracked(true);
    opts.recurse_untracked_dirs(true);

    let statuses = repo.statuses(Some(&mut opts))
        .map_err(|e| AppError::Git(format!("获取状态失败: {e}")))?;

    if statuses.is_empty() && dto.mid_operation.is_none() {
        return Ok(dto);
    }

    for entry in statuses.iter() {
        let s = entry.status();

        if s.contains(git2::Status::CONFLICTED) {
            dto.has_conflicts = true;
            dto.is_dirty = true;
            dto.changed_count += 1;
            continue;
        }

        if s.intersects(
            git2::Status::INDEX_NEW | git2::Status::INDEX_MODIFIED
            | git2::Status::INDEX_DELETED | git2::Status::INDEX_RENAMED
            | git2::Status::INDEX_TYPECHANGE,
        ) {
            dto.has_staged = true;
            dto.is_dirty = true;
            dto.changed_count += 1;
        }

        if s.intersects(
            git2::Status::WT_MODIFIED | git2::Status::WT_DELETED
            | git2::Status::WT_TYPECHANGE | git2::Status::WT_RENAMED,
        ) {
            dto.has_unstaged = true;
            dto.is_dirty = true;
        }

        if s.contains(git2::Status::WT_NEW) && !s.contains(git2::Status::INDEX_NEW) {
            dto.has_untracked = true;
            dto.is_dirty = true;
            dto.untracked_count += 1;
        }
    }

    if dto.mid_operation.is_some() {
        dto.is_dirty = true;
    }

    Ok(dto)
}
```

**性能考量**：对于大型仓库（>10000 文件），git2 的 statuses 较慢（libgit2 无 untracked-cache、单线程 lstat）。Phase 2 仓库规模可控，git2 足够；未来可降级到 `git status --porcelain` 子进程。

### 1.3 活跃终端会话检测

**规则**：若 worktree 中有正在运行的终端会话（CWD 属于该 worktree），必须在删除目录前 kill 会话，否则 Windows 文件锁会阻止删除。

**实现**：
- 前端已有 `getSessionsByWorktree(worktreeId)` 和 `getPanelIdsByWorktree(worktreeId)` 辅助函数。
- `SessionInfo.worktreeId` 在 spawn 返回后通过 `setSession` 写入。
- 删除前并行 kill 所有受影响会话：

```typescript
import { getSessionsByWorktree } from "../../stores/sessions";
import { pty } from "../../ipc";

async function killSessionsForWorktree(worktreePath: string): Promise<void> {
  const sessions = getSessionsByWorktree(worktreePath);
  if (sessions.length === 0) return;
  await Promise.all(sessions.map((s) => pty.kill(s.sessionId)));
}
```

**注意**：`pty_kill` 中的 `ClosePseudoConsole` 在 pre-Win11 24H2 上可能永久阻塞，但已在 `spawn_blocking` 中执行，不阻塞 IPC worker。前端不需要等待文件句柄释放——kill 后 OS 活动最终会释放锁。

### 1.4 关联操作页面检测

**规则**：删除 worktree 时，绑定到该 worktree 的所有操作页面应自动移除。

**实现**：
- 通过 `project.pages.filter(p => p.binding?.worktreePath === worktreePath)` 获取受影响页面。
- 名称列表传入确认对话框展示。
- 删除时由 `useWorktreeDelete` 统一处理页面移除和空页面回退。

### 1.5 路径有效性检测

**规则**：删除前验证路径存在且是有效的 git worktree。

**实现**：在 `git_worktree_remove` 内部，通过 `git worktree list --porcelain` 验证路径在列表中（而非仅检查 `.git` 文件存在，后者对非 worktree 子目录也可为 true）。

### 1.6 并发安全：每仓库锁

**背景**：多个前端操作可能同时操作同一仓库的 worktree（如同时 add + remove），需要锁机制避免竞态。

**模式**（来自 rsworktree / iso-code / claude-supervisor 的通用做法）：

```rust
use std::collections::HashMap;
use std::sync::{Arc, Mutex};

/// 每仓库锁：PathBuf → Arc<Mutex<()>>
pub struct RepoLockMap {
    locks: Mutex<HashMap<String, Arc<Mutex<()>>>>,
}

impl RepoLockMap {
    pub fn acquire(&self, repo_path: &str) -> Arc<Mutex<()>> {
        let mut locks = self.locks.lock().unwrap();
        locks
            .entry(repo_path.to_string())
            .or_insert_with(|| Arc::new(Mutex::new(())))
            .clone()
    }
}
```

在 `git_worktree_add`、`git_worktree_remove` 的入口获取对应仓库的锁，确保同一仓库的 worktree 增删操作串行化。

**放置位置**：`AppState` 或 `git` 模块的模块级 `static`。

## 2. 删除序列 (Deletion Sequence)

**重要前提**：libgit2 **没有** `git_worktree_remove` API。`git2::Worktree` 结构体有 `prune()` 但无 `remove()`——删除工作树的唯一可靠程式化方式是通过 shell 调用 `git worktree remove`。三级回退策略在此基础上设计，每级尝试前检查上一级是否已成功清理。

### 2.1 序列流程

```
Level 1: git worktree remove --force <rel_path>     (优雅删除: 清理元数据 + 目录)
  ├─ exit 0                  → 跳转 prune
  ├─ "not a working tree"    → 目录残留 → Level 2
  ├─ "分支 not found"         → 目录残留 → Level 2
  ├─ "file locked"           → 重试(最多2次, 500ms间隔)
  └─ 其他错误                 → 返回错误

Level 2: cmd.exe /c (attrib -R "<path>\*" /S /D && rmdir /S /Q "<path>")
  ├─ rmdir /S /Q 不穿透 NTFS junction 点（安全）
  ├─ 成功                     → 验证目录不存在 → prune
  └─ 失败                     → 返回错误(文件占用/权限)

Level 3: remove_dir_all crate（处理只读/交接点/长路径，并行加速）
  ├─ 推荐使用 `remove_dir_all` crate v0.8（非 std::fs::remove_dir_all）
  ├─ std 在 Rust<1.71 对只读文件报 PermissionDenied，git 文件默认只读
  ├─ 安全保证: 仅当前两个 Level 失败目录仍存在时才执行（而非首选手法）
  ├─ 注意: 自定义 Tauri 命令无插件 scope 保护——路径须经规范化+范围校验
  ├─ 成功                     → prune
  └─ 失败                     → 返回最终错误（含指数退避重试）

prune: git worktree prune                           (清理元数据残留)
```

### 2.2 核心实现

```rust
// src-tauri/src/git/worktree.rs

/// 删除 worktree — 三级回退 + 重试 + prune 清理
#[tauri::command]
pub async fn git_worktree_remove(
    repo_path: String,
    name: String,
) -> Result<WorktreeRemoveResult, AppError> {
    let sanitized = sanitize_branch_name(&name);
    if sanitized.is_empty() {
        return Err(AppError::Worktree("分支名不能为空".to_string()));
    }

    let worktree_rel = format!(".claude/worktrees/{sanitized}");
    let worktree_abs = format!(
        "{}\\{}",
        repo_path.trim_end_matches('\\').trim_end_matches('/'),
        worktree_rel.replace('/', "\\")
    );

    // 前置验证: 不是主工作树
    validate_worktree_is_not_main(&repo_path, &worktree_abs)?;

    // 重试配置
    const MAX_RETRIES: u32 = 2;
    const RETRY_DELAY_MS: u64 = 500;

    // ── Level 1: git worktree remove --force ──
    let git_result = try_git_worktree_remove(&repo_path, &worktree_rel, MAX_RETRIES, RETRY_DELAY_MS);

    // ── Level 2: cmd rmdir (仅当目录仍存在) ──
    if std::path::Path::new(&worktree_abs).exists() {
        try_cmd_rmdir(&worktree_abs)?;
    }

    // ── Level 3: Rust remove_dir_all (long path prefix, 仅当目录仍存在) ──
    if std::path::Path::new(&worktree_abs).exists() {
        let long_path = format!("\\\\?\\{}", worktree_abs);
        let path = long_path.clone();
        tokio::task::spawn_blocking(move || remove_dir_all_robust(&path))
            .await
            .map_err(|e| AppError::Worktree(format!("spawn_blocking 失败: {e}")))?;
    }

    // 最终验证
    if std::path::Path::new(&worktree_abs).exists() {
        return Err(AppError::Worktree("删除失败: 目录仍然存在, 可能被占用".to_string()));
    }

    // 清理 git 元数据
    let _ = std::process::Command::new("git")
        .args(["worktree", "prune"])
        .current_dir(&repo_path)
        .output();

    Ok(WorktreeRemoveResult { success: true, path: worktree_abs })
}

/// 验证不是主工作树: 规范化路径后与仓库根目录比较
fn validate_worktree_is_not_main(repo_path: &str, worktree_path: &str) -> Result<(), AppError> {
    let repo = git2::Repository::open(repo_path)
        .map_err(|e| AppError::Git(format!("无法打开仓库: {e}")))?;
    let main_path = repo.workdir()
        .ok_or_else(|| AppError::Git("仓库无工作目录".to_string()))?;

    let worktree_canon = std::path::Path::new(worktree_path)
        .canonicalize()
        .unwrap_or_else(|_| std::path::PathBuf::from(worktree_path));
    let main_canon = main_path
        .canonicalize()
        .unwrap_or_else(|_| main_path.to_path_buf());

    if worktree_canon == main_canon {
        return Err(AppError::Worktree("不能删除主工作树".to_string()));
    }
    Ok(())
}

/// Windows: cmd.exe attrib -R + rmdir /S /Q（不穿透 junction 点）
#[cfg(windows)]
fn try_cmd_rmdir(path: &str) -> Result<(), AppError> {
    // 移除只读属性
    let _ = std::process::Command::new("cmd.exe")
        .args(["/c", &format!("attrib -R \"{}\\*\" /S /D", path)])
        .output();

    // 删除目录树（rmdir 不穿透 junction，安全）
    let output = std::process::Command::new("cmd.exe")
        .args(["/c", &format!("rmdir /S /Q \"{}\"", path)])
        .output()
        .map_err(|e| AppError::Worktree(format!("rmdir 失败: {e}")))?;

    if !output.status.success() {
        return Err(AppError::Worktree("文件被占用或权限不足".to_string()));
    }
    Ok(())
}

/// 健壮目录删除: 使用 remove_dir_all crate v0.8（处理只读/交接点安全）
/// crate 自动处理: 只读属性移除、长路径（\\?\）、NTFS 交接点安全（RUSTSEC-2023-0018 已修复）
fn remove_dir_all_robust(path: &str) -> Result<(), AppError> {
    let p = std::path::Path::new(path);
    if !p.exists() {
        return Ok(());
    }
    // remove_dir_all crate v0.8+ 自动处理只读属性和交接点安全
    // 注意：需 Cargo.toml 添加 `remove_dir_all = "0.8"`
    remove_dir_all::remove_dir_all(p)
        .map_err(|e| AppError::Worktree(format!("remove_dir_all 失败: {e}")))?;
    Ok(())
}

fn remove_readonly_attr(path: &std::path::Path) {
    if let Ok(meta) = std::fs::metadata(path) {
        let mut perms = meta.permissions();
        if perms.readonly() {
            perms.set_readonly(false);
            let _ = std::fs::set_permissions(path, perms);
        }
    }
    if path.is_dir() {
        if let Ok(entries) = std::fs::read_dir(path) {
            for entry in entries.flatten() {
                remove_readonly_attr(&entry.path());
            }
        }
    }
}
```

### 2.3 重试策略

**关键 Windows 纠正（来自 agent ab0f0b9 研究）**：`ERROR_SHARING_VIOLATION` (32) 和 `ERROR_LOCK_VIOLATION` (33) 在 Rust std 中映射为 `ErrorKind::Other`，**不是** `PermissionDenied`。检测可重试错误必须通过 `err.raw_os_error()` 而非 `err.kind()`。

推荐使用 `backoff` crate 的指数退避（带 jitter），而非简单固定延迟循环：

```toml
# Cargo.toml 新增
backoff = { version = "0.4", features = ["tokio"] }
```

```rust
use backoff::{ExponentialBackoff, Error as BackoffError};

fn is_retryable(err: &std::io::Error) -> bool {
    #[cfg(windows)]
    {
        // ERROR_SHARING_VIOLATION=32, ERROR_LOCK_VIOLATION=33,
        // ERROR_ACCESS_DENIED=5（可能是临时文件锁）, ERROR_DIR_NOT_EMPTY=145
        matches!(err.raw_os_error(), Some(5 | 32 | 33 | 145))
    }
    #[cfg(not(windows))]
    {
        matches!(err.kind(), std::io::ErrorKind::WouldBlock | std::io::ErrorKind::TimedOut)
    }
}

fn try_remove_with_retry(path: &str) -> Result<(), AppError> {
    let backoff = ExponentialBackoff {
        initial_interval: Duration::from_millis(50),
        max_interval: Duration::from_secs(2),
        max_elapsed_time: Some(Duration::from_secs(10)),
        multiplier: 2.0,
        randomization_factor: 0.5, // ±50% jitter，避免惊群
        ..ExponentialBackoff::default()
    };

    let p = PathBuf::from(path);
    let result = backoff::retry(backoff, || {
        match std::fs::remove_dir_all(&p) {
            Ok(()) => Ok(()),
            Err(e) if is_retryable(&e) => Err(BackoffError::transient(e)),
            Err(e) => Err(BackoffError::permanent(e)),
        }
    });

    result.map_err(|e| AppError::Worktree(format!("删除失败(已重试): {e}")))
}
```

| 阶段 | 策略 | 参数 |
|------|------|------|
| git worktree remove | 固定延迟重试 | max 2 次, 500ms 间隔 |
| cmd rmdir | 单次（不重试） | - |
| Rust remove_dir_all | 指数退避 + jitter | 50ms→2s, max 10s |

### 2.4 `git worktree remove` 预期退出码

| 退出码 | stderr 关键词 | 含义 | 处理 |
|--------|-------------|------|------|
| 0 | (空) | 成功 | 跳转 prune |
| 128 | `not a working tree` | 路径非 worktree | 降级至 Level 2 |
| 128 | `is a main working tree` | 主工作树 | 返回错误（不应出现，前端已挡） |
| 128 | `contains modified or untracked files` | 有未提交更改(无 --force) | 有 --force 不应触发 |
| 1 | `cannot lock ref` | 引用锁定 | 重试 |
| 1 | `unable to unlink` | 文件权限 | 降级至 Level 2 |
| 128 | `not a git repository` | 仓库损坏 | 返回错误 |

## 3. Store 清理

### 3.1 前端完整删除流程

```
用户点击删除 → 预检 confirm dialog → 确认:

1. 调用 git.getWorktreeStatus(worktreePath) → 展示在对话框
2. 用户勾选 checkbox gate
3. 获取受影响 sessions (getSessionsByWorktree)
4. 并行 pty.kill 所有受影响 session
5. store.removeWorktree(projectId, worktreePath)
6. 遍历受影响页面: store.removePage(projectId, pageId)
7. 若项目无剩余页面: 创建默认空页面(cwd=仓库根目录)
8. 调用 git.removeWorktree(repoPath, name) ← 后端三级回退
9. debounce 2s 后自动 persist
```

### 3.2 活动页面自动切换

`removePage` 已有自动切换逻辑（`src/stores/projects.ts:128-139`）：若移除的是当前活动页面，自动切换到 `pages[0]`；若全部页面被移除，需额外处理——创建默认空页面避免空白工作区。

### 3.3 封装函数

```typescript
// src/features/worktree/useWorktreeDelete.ts

export async function deleteWorktree(params: {
  projectId: string;
  repoPath: string;
  worktreeName: string;
  worktreePath: string;
}): Promise<void> {
  const { projectId, repoPath, worktreeName, worktreePath } = params;
  const store = useProjects.getState();
  const project = store.projects[projectId];
  if (!project) throw new Error("项目不存在");

  // 1. Kill 所有受影响终端会话
  const sessions = getSessionsByWorktree(worktreePath);
  if (sessions.length > 0) {
    await Promise.all(sessions.map((s) => pty.kill(s.sessionId)));
  }

  // 2. Store: 移除 worktree
  store.removeWorktree(projectId, worktreePath);

  // 3. Store: 移除所有绑定页面
  const affectedPages = project.pages.filter(
    (p) => p.binding?.worktreePath === worktreePath
  );
  for (const page of affectedPages) {
    store.removePage(projectId, page.pageId);
  }

  // 4. 若无剩余页面, 创建默认空页面
  const updated = useProjects.getState().projects[projectId];
  if (updated && updated.pages.length === 0) {
    store.addPage(projectId, {
      pageId: createPageId(),
      name: "默认",
      layout: {},
      createdAt: Date.now(),
      lastAccessedAt: Date.now(),
    });
  }

  // 5. 后端清理
  await git.removeWorktree(repoPath, worktreeName);
}
```

## 4. 错误分类体系 (Error Taxonomy)

### 4.2 错误码 → 用户消息映射表

| 错误场景 | 用户消息（中文） | 建议操作 | 展示方式 |
|----------|-----------------|---------|---------|
| 目录不存在 / 非 worktree | "路径不是有效的工作树" | 检查路径是否正确，或已手动删除 | Toast (error) |
| 主工作树 | "不能删除主工作树" | 按钮不应出现（前端已挡） | 按钮置灰 |
| 有未提交更改 | "工作树有 {n} 个文件未提交" | commit/stash 或勾选确认 | 对话框警告 |
| 有进行中操作 | "工作树正在进行 {op}，请先完成或中止" | 完成操作或中止 | 对话框阻断 |
| 权限拒绝 | "无法删除工作树：权限不足" | 检查文件是否被其他程序打开 | Toast + 重试按钮 |
| 文件被占用 | "无法删除工作树：文件被占用" | 关闭占用程序后重试 | Toast + 重试按钮 |
| 磁盘/IO 错误 | "删除失败：磁盘或系统错误" | 检查磁盘空间 | Toast |
| Git 元数据损坏 | "工作树 Git 元数据已损坏" | 可手动删除目录 + `git worktree prune` | Toast + 手动指引 |
| 路径过长 | "路径过长，删除失败" | 缩短路径或手动删除 | Toast |
| 重试耗尽 | "删除失败：已重试多次仍无法删除" | 显示路径，手动清理 | Dialog + 路径复制 |

### 4.3 前端错误转译

```typescript
// src/ipc/git.ts

export class WorktreeDeleteError extends Error {
  constructor(public code: string, message: string) {
    super(message);
  }
}

export async function removeWorktree(repoPath: string, name: string) {
  try {
    return await invoke("git_worktree_remove", { repoPath, name });
  } catch (err) {
    const msg = String(err);
    if (msg.includes("不能删除主工作树"))
      throw new WorktreeDeleteError("MAIN_WORKTREE", "不能删除主工作树");
    if (msg.includes("Permission denied") || msg.includes("权限不足"))
      throw new WorktreeDeleteError("PERMISSION_DENIED", "无法删除：权限不足");
    if (msg.includes("文件被占用"))
      throw new WorktreeDeleteError("FILE_IN_USE", "无法删除：文件被占用");
    if (msg.includes("不是有效的工作树"))
      throw new WorktreeDeleteError("INVALID_WORKTREE", "路径不是有效的工作树");
    throw new WorktreeDeleteError("UNKNOWN", `删除失败: ${msg}`);
  }
}
```

### 4.4 错误展示策略

| 严重性 | 展示方式 | 场景 |
|--------|---------|------|
| 阻断 | 对话框内红色警告，禁用确认按钮 | 主工作树、进行中操作 |
| 警告 | 对话框内黄色提示 + checkbox gate | 有未提交更改 |
| 错误 (可恢复) | Toast + 重试按钮 | 权限拒绝、文件占用 |
| 致命 | Dialog + 路径复制 + 手动修复指引 | 重试耗尽、元数据损坏 |

### 5.2 L1: Command 的模拟

`std::process::Command` 不可直接 mock。两种方案：

**方案 A：Trait 抽象（推荐用于需要精确控制 exit code/stderr 的场景）**

```rust
/// Git 命令执行器 trait — 生产用 RealGitRunner, 测试用 Mock
pub trait GitCommandRunner: Send + Sync {
    fn run(&self, args: &[&str], cwd: &str) -> std::io::Result<std::process::Output>;
}

pub struct RealGitRunner;
impl GitCommandRunner for RealGitRunner {
    fn run(&self, args: &[&str], cwd: &str) -> std::io::Result<std::process::Output> {
        std::process::Command::new("git").args(args).current_dir(cwd).output()
    }
}
```

**方案 B：真实临时仓库（当前项目已采用，适合集成场景）**

现有 `test_worktree_list_add_remove` 测试用 `tempfile::tempdir()` + `git init` 执行真实流程。此方案适合 L3 集成测试。

**建议**：L3 用方案 B（真实 git 仓库），若需 L1 单元测试精确控制 exit code 则引入方案 A。

### 5.3 L3 集成测试用例

```rust
#[test]
fn test_worktree_status_clean_and_dirty() {
    let (repo_path, _temp) = setup_temp_repo();
    // 新建 worktree → 应干净
    let wt = git_worktree_add(repo_path.clone(), "test-clean".into()).unwrap();
    let status = git_worktree_status(wt.path).unwrap();
    assert!(!status.is_dirty);

    // 制造未追踪文件
    std::fs::write(format!("{}/dirty.txt", wt.path), "x").unwrap();
    let status = git_worktree_status(wt.path).unwrap();
    assert!(status.is_dirty);
    assert!(status.has_untracked);
    assert_eq!(status.untracked_count, 1);
}

#[test]
fn test_worktree_remove_dirty_then_clean() {
    let (repo_path, _temp) = setup_temp_repo();
    let wt = git_worktree_add(repo_path.clone(), "test-rem".into()).unwrap();
    // dirty 删除（--force 应允许）
    std::fs::write(format!("{}/dirty.txt", wt.path), "x").unwrap();
    let result = git_worktree_remove(repo_path.clone(), "test-rem".into());
    assert!(result.is_ok());
    assert!(!std::path::Path::new(&wt.path).exists());
}

#[test]
fn test_worktree_remove_main_blocked() {
    let (repo_path, _temp) = setup_temp_repo();
    // 用仓库根目录路径（主工作树）构造删除请求 → 应被阻断
    let result = git_worktree_remove(repo_path.clone(), "..".into());
    // 或者通过 validate_worktree_is_not_main 直接测试
    assert!(result.is_err());
}
```

### 5.4 边缘用例清单

| 用例 | 场景 | 预期行为 |
|------|------|---------|
| 并发删除 | 两个前端实例同时删除同一 worktree | 第一个成功，第二个返回"路径不存在" |
| CWD 锁 | 终端 CWD 在要删的目录内 | rmdir 失败 → 提示"文件被占用" |
| 只读文件 | worktree 含只读 .git/objects | attrib -R 或 remove_readonly_attr 后成功 |
| 长路径 | 绝对路径 > 260 字符 | `\\?\` 前缀绕过 MAX_PATH |
| Junction 点 | worktree 内含 NTFS junction | rmdir 不穿透，安全 |
| 分支已手动删除 | `git branch -D worktree-xxx` 后 remove | git remove 失败 → 降级到文件清理 |
| .git 元数据损坏 | worktree 的 .git 文件指向无效路径 | git remove 失败 → 降级到文件清理 → prune |
