## 关键设计决策

### H6：3 条路径验证

- **页面切换**（isLayoutSwitching=true）：不 dispose term/addon，存回 Registry
- **用户关面板**（isLayoutSwitching=false）：正常 dispose + Registry.remove + kill PTY
- **删除页面**（S1 修复后 isLayoutSwitching=false）：全部正常销毁

Phase 2 务实方案：Term 实例存活（DOM/WebGL 可复用），切回时重新 spawn PTY（旧 Channel 随 reader 线程退出）。Channel 重连留 Phase 3。

### H6 A5：已就绪，跳过

全部 6 个终端入口均已设置 `renderer: 'always'`：Watermark(:73) / RightHeaderActions(:125) / onReady(:261) / 右键菜单(:303) / makeDefaultLayout / panelRegistry 默认值。

### S1：onDeletePage 区分场景 A/B

```typescript
const onDeletePage = useCallback((projectId: string, pageId: string) => {
  const api = apiRef.current;
  if (!api) return;

  const layoutStore = useLayout.getState();
  const isActive = layoutStore.activePageId === pageId;

  if (isActive) {
    flushDirtyLayout();
    api.clear();
    layoutStore.setActivePage(null);
  }

  useProjects.getState().removePage(projectId, pageId);

  if (isActive) {
    const nextPageId = useProjects.getState().projects[projectId]?.activePageId;
    if (nextPageId) {
      switchToPage(projectId, nextPageId);
    }
  }
}, [flushDirtyLayout, switchToPage]);
```

### S2：PageRow 内联编辑

PageRow props 新增 `onRename: (newName: string) => void`。父组件传 `(newName) => renamePage(projId, page.pageId, newName)`。PageRow 内部封装 editing/editValue state + `<input>`（autoFocus、Enter 确认、Escape 取消、onBlur 确认）。

### S4：启动时序合并

单一 useEffect，消除隐式竞态：
```
loadAllProjects() → markPersistenceReady() → setActivePage(lastPage) → setReady(true) → render <Workspace />
```

`Workspace.onReady` 执行时 store 数据已就绪 → 移除 `isReadyRef` 重试防护（`restoreGuardRef` 保留——fromJSON 恢复守卫仍需）。
