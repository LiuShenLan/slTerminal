## 2. 人工验收未通过项

### 2.1 H6：操作页面切换空白循环 + 终端全新启动

| 文件 | 行号 | 根因 |
|------|------|------|
| `Workspace.tsx` | 158 | 无自切换守卫：`if (layoutStore.isLayoutSwitching) return` 但无 `if (activePageId === pageId) return` |
| `SidebarTree.tsx` | 305, 328 | 新页面默认 `layout: {}`（空对象）→ `switchToPage` 的 `Object.keys({}).length > 0` 为 false → 走 `api.clear()` 分支 → 空白态 |
| `Workspace.tsx` | 223-226 | `onDidLayoutChange` 回调仅 `console.debug`，不写 store → 当前页布局从不持久化 |
| `useXterm.ts` | 243, 246 | cleanup 无条件 `pty.kill()` + `term.dispose()` → Dockview `fromJSON` 内部 `clear()` 触发面板卸载 → 销毁 PTY → 切回时全新 spawn |
| Dockview 源码 | `dockviewComponent.js:1164-1186` | `fromJSON` 内部 `clear()` 销毁所有 ID 不匹配面板 → `reuseExistingPanels` 保护的仅是 JSON 中仍存在的 ID |

**关键子问题链**：

| 子问题 | 根因 |
|--------|------|
| 空白→恢复循环 | 空布局 `{}` 与有内容布局之间反复切换，`api.clear()` / `fromJSON` 交替触发 |
| 恢复后终端是全新的 | 面板 ID 跨页面不同（`terminal-{Date.now()}`），`reuseExistingPanels` 无命中 → 全部销毁重建 |
| 终端无之前操作记录 | 旧 Terminal `dispose()` 不可逆 + 新 `pty.spawn()` 创建全新 shell 进程 |

### 2.2 H7：重启后项目恢复但布局不恢复

| 文件 | 行号 | 根因 |
|------|------|------|
| `Workspace.tsx` | 223-226 | `onDidLayoutChange` 仅 `console.debug`，当前页布局从不写入 store → 持久化文件始终保持 `layout: {}` |
| `Workspace.tsx` | 211-235 | `onReady` 不恢复活跃页面布局：只检查 projects 是否为空来创建默认终端，不读取 `activePageId` 对应的 `page.layout` 并调用 `loadLayout` |
| `App.tsx` | 35-40 | `loadAllProjects()` 是 async，`onReady` 先触发时 store 为空 → 无终端创建；`loadAllProjects` 完成后无代码触发布局恢复 |
| 全局 | — | 无 `onCloseRequested` / `beforeunload` 关闭钩子——仅靠 Zustand subscribe 2s debounce。kill/崩溃/关机时 debounce 窗口内数据丢失 |

**排除**：非 Dockview `toJSON/fromJSON` bug（源码确认机制正确）、非 `params` 序列化问题、非路径编码问题

### 2.3 N1：删除 worktree 不生效

| 文件 | 行号 | 根因 |
|------|------|------|
| `SidebarTree.tsx` | 400 | 前端传 `worktree.branch`（如 "main"） |
| `git/worktree.rs` | 113-115 | 后端从分支名重建路径 `format!(".claude/worktrees/{sanitized_branch}")` → 主 worktree 路径为仓库根目录而非 `.claude/worktrees/main` → `git worktree remove` 找不到路径 → 失败 |
| `SidebarTree.tsx` | 399 | 仅 `window.confirm`，未导入 `DeleteWorktreeConfirm` 组件 |
| `SidebarTree.tsx` | 458 | 错误仅 `console.error`，无用户可见反馈 |

**附加发现**：
- 主 worktree 未从删除菜单排除——用户可点击"删除工作树"对主 worktree 执行（虽然会失败但不应出现此选项）
- 删除前无脏工作树检查（行业标准：JetBrains 阻止，GitKraken 警告）
- 无 CWD 检测——终端进程 CWD 在 worktree 内时 `rmdir` 可能失败

### 2.4 N2：终端会话不保留

| 文件 | 行号 | 根因 |
|------|------|------|
| `useXterm.ts` | 243 | `pty.kill(sessionId)` 在 cleanup 中无条件执行 |
| `useXterm.ts` | 246 | `term.dispose()` 不可逆——xterm.js `open()` 是 one-shot |
| `Workspace.tsx` | 168-185 | `switchToPage` → `fromJSON` 内部 `clear()` → 面板卸载 → React cleanup 触发 → kill + dispose |

**架构层面**：Tauri `Channel<PtyEvent>` 绑定到单次 `invoke` 调用。即使不 kill PTY，前端也无法跨组件挂载复用 Channel。完整方案需新增 `pty_reattach(sessionId, channel)` 命令。

**行业对比**：
- VS Code：`PtyHost` 独立进程 + `TerminalProcessManager` 状态机 + detach/reattach
- WezTerm：`mux-server` daemon + `wezterm connect` CLI reattach
- Tabby：RxJS `Observable<Buffer>` 解耦 Session 与 Frontend

```
Step 1: N1 删除 worktree (独立)

          │   └→ 需 TerminalRegistry fallback spawn
          └→ Step 5: H6 默认布局 + 自切换守卫
```
