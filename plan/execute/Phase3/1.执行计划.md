#### A3 — 实现 notify 文件监听模块

**背景**：`notify` 模块为占位符，notify crate 已安装但未使用。

**经 3 agent 检索确认**：
- notify 9.0.0-rc.4 是第 4 个候选版，已足够稳定
- 需追加 `notify-debouncer-full`（官方去抖 + FileIdCache + rename 缝合）
- Windows 上 `ReadDirectoryChangesW` 已知限制：网络驱动器不可靠、重复事件、需处理 `need_rescan()` 标志

```rust
// notify/mod.rs
use notify_debouncer_full::{new_debouncer, DebouncedEvent, RecommendedCache};
use std::sync::mpsc;
use tauri::Emitter;  // 用 Event 系统广播 FsEvent（非 Channel——事件需广播给多个面板）

// fs_watch(path) → 启动后台 watcher 线程
//   去抖 300-500ms（notify-debouncer-full 内置）
//   通过 app_handle.emit("fs-event", payload) 广播事件
//   返回 watch_id 供后续 unwatch
//
// fs_unwatch(watch_id) → 停止监听
```

**Windows 特殊处理**：
- 网络驱动器检测失败时降级到 PollWatcher（2s 轮询间隔）
- `need_rescan()` 事件发特殊标记通知前端全量刷新
- 事件去抖：300ms（`notify-debouncer-full` 默认值）

### 任务组 E：PTY 重连（来自 note.md，依赖现有 PTY 模块）

#### E1 — Ring buffer + `pty_reattach` 命令

**Rust 端改造**（`pty/spawn.rs` + `state.rs`）：

```rust
// state.rs — PtySession 扩展
pub struct PtySession {
    // ... 现有字段
    pub channel: Arc<RwLock<Option<Channel<PtyEvent>>>>,  // 可替换 Channel
    pub output_ring: Arc<Mutex<VecDeque<u8>>>,            // 64KB ring buffer
    pub reader_handle: Option<JoinHandle<()>>,
}

// pty/spawn.rs — 新命令
#[tauri::command]
fn pty_reattach(
    state: tauri::State<'_, AppState>,
    session_id: String,
    channel: Channel<PtyEvent>,
) -> Result<(), AppError> {
    let sessions = state.pty.sessions.read().unwrap();
    let session = sessions.get(&session_id).ok_or(AppError::SessionNotFound)?;

    // 1. 替换 Channel
    let mut ch = session.channel.write().unwrap();
    *ch = Some(channel);

    // 2. 回放 ring buffer 内容
    let ring = session.output_ring.lock().unwrap();
    if !ring.is_empty() {
        let data: Vec<u8> = ring.iter().copied().collect();
        if let Some(ref c) = *ch {
            let _ = c.send(PtyEvent::Output { bytes: data });
        }
    }
    Ok(())
}
```

**Reader 线程改造**：
- `channel.send()` 失败时不退出线程，仅写入 ring buffer
- Ring buffer 固定 64KB，FIFO（`VecDeque::pop_front` 腾空间）
- 孤儿超时：300s 无重连 → kill PTY（Tauri Event 或定时器检测）

**前端侧**（`src/ipc/pty.ts` + `useXterm.ts`）：

```typescript
// pty.ts — 新增
export async function reattach(sessionId: string, onOutput: callback): Promise<void> {
  const channel = new Channel<PtyEvent>();
  channel.onmessage = (msg: PtyEvent) => {
    if (msg.type === "output") onOutput(new Uint8Array(msg.data.bytes));
  };
  await invoke("pty_reattach", { sessionId, channel });
}

// useXterm.ts — mount 时逻辑
// 1. 查 TerminalRegistry → 有已有 sessionId
// 2. 调 pty.reattach(sessionId, onOutput) 替代 pty.spawn()
// 3. Ring buffer 回放自动恢复之前的终端内容
```

```
Phase 6: D1–D3 并行（依赖 B3 + C1）
Phase 7: E1 + E2 并行（依赖现有 PTY 模块）
Phase 8: 测试与验收
```

## 8. 注意事项

1. **git2 的 `vendored-libgit2` feature 是必须的**——否则 Windows 上编译报 `LNK1181: cannot open input file 'git2.lib'`
2. **notify Windows 重复事件**——单次保存触发 2-4 次事件，必须用 `notify-debouncer-full` 去抖
3. **notify `need_rescan()` 标志**——缓冲区溢出时必须通知前端全量刷新
4. **CM6 StateField 的 `map` 方法**——必须正确实现以跟随文档编辑保持 marker 位置同步
5. **Ctrl+F 与 prevent-default**——Phase 2 修复已豁免 FIND flag，不会冲突
6. **git worktree 永久舍弃**——不实现 worktree 创建/列表/删除，文件浏览器中过滤 `.claude/worktrees/`
7. **PTY 重连的孤儿超时**——300s 无重连则 kill PTY，防止资源泄漏
8. **配色 token 反硬编码**——文件浏览器和 diff 边栏必须引用 `theme/colors.ts`，禁止内联颜色值
