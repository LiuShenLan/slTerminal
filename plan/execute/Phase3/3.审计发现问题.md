## 2. 人工验收失败项根因

### 2.1 🔴 FileWatcher 未启动（影响 X1、X2）

**现象**：
- X1：外部修改文件后编辑器内容不自动重载
- X2：编辑器有未保存修改时外部修改文件，无弹窗提示

**根因**：

`src-tauri/src/notify/mod.rs` 实现了完整的 `FileWatcher` 结构体（`start()`/`stop()`/`Drop`、去抖、`need_rescan` 处理），但**从未集成到应用生命周期中**：

1. `AppState`（`state.rs:35-37`）不含 `FileWatcher` 字段——只有 `pty: PtyState`
2. `lib.rs::run()` 未创建 `FileWatcher`、未调用 `start()`
3. 没有 `fs_watch` 命令注册在 `generate_handler!` 中
4. 前端 `useFileTree.ts:160-176` 和 `useCodeMirror.ts:148-187` 订阅了 Tauri event `"fs-event"`，但后端**无人发射此事件**

**整条 notify 管线断链**：watcher 未启动 → 无事件 → 前端 handler 永不触发 → X1/X2 永远不生效。

**涉及文件**：
- `src-tauri/src/state.rs` — AppState 缺 FileWatcher 字段
- `src-tauri/src/lib.rs` — run() 未初始化 FileWatcher，缺 fs_watch 命令注册
- （可选）`src-tauri/src/notify/mod.rs` — 代码完整无需修改

### 2.2 🔴 git_status 路径不匹配（影响 F5、V4）

**现象**：
- F5：文件浏览器中文文件名无 git 状态着色
- V4：文件名颜色与 `git status` 输出不一致

**根因**：路径表示不一致导致 `gitStatusMap` 查找永远 miss。

- `fs_read_dir`（`fs/mod.rs`）返回的 `DirEntry.path` 是**绝对路径**：`D:\data\learn\code\slTerminal\src\main.tsx`
- `git_status`（`git/mod.rs:56-57`）通过 `entry.path().unwrap().to_string()` 返回 `git2::StatusEntry::path()`，该路径是**相对仓库根的路径**：`src/main.tsx`

`useFileTree.ts` 中：
1. L103 `gitStatusMap.get(child.entry.path)` — key 为绝对路径，map 内 key 为相对路径 → 永远 miss
2. L63-69 `loadRoot()` 调用 `loadDirectory()` 时完全不设置 `gitStatus` 字段，即使路径匹配也需额外 useEffect 对 rootNodes 补上 gitStatus

**涉及文件**：
- `src/features/explorer/useFileTree.ts` — 路径归一化 + rootNodes 补 gitStatus
- 或 `src-tauri/src/git/mod.rs` — git_status 返回绝对路径（需知道仓库根）

### 2.3 🔴 git_diff 路径不匹配（影响 E4、V3）

**现象**：
- E4：编辑器左侧 diff 边栏无标记
- V3：diff 边栏无新增/修改/删除行着色

**根因**：双层路径错误。

**前端**（`useCodeMirror.ts:112-117`）：
```typescript
const parentDir =
    filePath.lastIndexOf("/") >= 0
      ? filePath.slice(0, filePath.lastIndexOf("/"))
      : ".";
const loadedHunks = await gitDiff(parentDir, filePath);
```
传入 `gitDiff` 的第一个参数是**文件所在父目录**，而非 git 仓库根。Windows 上若 `filePath` 使用反斜杠（`\`），`lastIndexOf("/")` 返回 -1 → `parentDir = "."`。

**后端**（`git/mod.rs:138-143`）：
```rust
let rel_path_str = file_path_std
    .strip_prefix(repo_path_std)
    .unwrap_or(file_path_std)
    .to_string_lossy()
    .replace('\\', "/");
opts.pathspec(&rel_path_str);
```
`pathspec` 应当相对 git 仓库根。若 `repo_path` 是文件父目录（非仓库根），则 `strip_prefix` 后得到短路径（如 `"main.tsx"`），而 git2 pathspec 期望完整相对路径（如 `"src/main.tsx"`）→ 0 hunks。

**涉及文件**：
- `src/panels/editor/useCodeMirror.ts` — 传入正确的 repo root 而非 parentDir
- 或 `src-tauri/src/git/mod.rs` — git_diff 自行从 file_path 向上查找 repo root
